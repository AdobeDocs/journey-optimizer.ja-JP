# 決定データセット {#decisions-dataset}

オファーが変更されるたびに、判定用の自動生成データセット（旧称：アクティビティ）が更新されます。

![](../../assets/dataset-activities.png)

正常に更新された、データセットの最新のバッチが右側に表示されます。データセットのスキーマの階層表示が左ウィンドウに表示されます。

>[!NOTE]
>
>オファーライブラリの各オブジェクト用に書き出したデータセットにアクセスする方法については、[この節](../export-catalog/access-dataset.md)を参照してください。

以下に、**[!UICONTROL 決定オブジェクトリポジトリ - 決定]**（旧称は「決定オブジェクトリポジトリ - アクティビティ」）データセットで使用できるすべてのフィールドの一覧を示します。

**エクスペリエンス/判定/条件/説明**

**フィールド：** 説明
**タイトル：** 説明
**説明：** 条件の説明。この条件の構成や目的、この条件が決定に与える影響などについて、人間が理解できる形で意図を伝えるために使用します。**型：**&#x200B;文字列

**_エクスペリエンス/判定/条件/オプション選択**

**フィールド：** optionSelectionタイ
**トル：** オプション選択
**説明：** オプション選択は、このコンテキストでのオプションの有効性/適用可能性を定義します。**型：**&#x200B;オブジェクト

* **説明**

   **フィールド：** description
   **タイトル：**説明
   **説明：**オプション選択の説明。このオプション選択の構成や目的、合致するオプションなどについて、人間が理解できる形で意図を伝えるために使用します。

* **オプションフィルター**

   **フィールド：**filter
   **タイトル：**オプションフィルター
   **説明：**付加されたタグを使用してインベントリ内のオプションと照合するタグベースのフィルターへの参照。値は、参照される決定ルールの URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/filter を参照してください。

   **型：**文字列
   **可能な値：** &quot;none&quot;（デフォルト）、&quot;directList&quot;、&quot;filter&quot;

* **オプションリスト**

   **フィールド：**options
   **タイトル：**オプションリスト
   **説明：**フィルタークエリを評価せずにオプションを直接指定するリスト。オプションリストかオプションフィルタールールのどちらかを指定できます。
   **型：**&#x200B;配列

**_エクスペリエンス/判定/条件/配置**

**フィールド：** 配置
**タイトル：** 配置制限
**説明：** 配置制約は、この条件がリストに表示されている配置にのみ適用できることを示します。対象となるプレースメントが `xdm:placements` リスト内にある場合にのみ、オプションの選択が考慮されます。それ以外の場合は、決定条件全体がスキップされます。「xdm:placements」リストが省略された、または空の場合、ターゲットとする任意のプレースメントに対して条件が考慮されます。ここに示すプレースメントは、オプション選択に対して暗黙の条件を課します。考慮するオプションには、ターゲットとするプレースメントの表現が必要です。**型：**&#x200B;配列

* **プレースメント識別情報**

   **タイトル：**プレースメント識別情報
   **説明：**プレースメントエンティティに対する参照。値は、参照されるプレースメントの URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/placement を参照してください。
   **型：**&#x200B;文字列

**_エクスペリエンス/判定/条件/profileConstraints**

**フィールド：** profileConstraints
**タイトル：** プロファイル制約説
**明：** プロファイル制約は、このコンテキストにおいて、オプション選択が現時点でこのプロファイルIDに対して適格かどうかを決定します。プロファイルの制約で各オプションの値を考慮する必要がない場合、つまり、オプションの選択肢が不変である場合、プロファイルの制約が「false」と評価されると、オプション選択全体がキャンセルされます。一方、オプションをパラメーターとするプロファイルの制約ルールは、オプション選択で、オプションを選定するたびに評価されます。**型：**&#x200B;オブジェクト

* **_エクスペリエンス/判定/条件/profileConstraints/説明**

   **フィールド：** description
   **タイトル：**説明
   **説明：**プロファイルの制約の説明。このプロファイルの制約が設けられた経緯や理由、含まれるオプションや除外されるオプションは何かについて、人間が判読できる形で意図を伝えるために使用します。
   **型：**&#x200B;文字列

* **_エクスペリエンス/判定/条件/ profileConstraints /実施要件ルール**

   **フィールド：** eligibilityRule
   **タイトル：**実施要件ルール
   **説明：**特定のプロファイルや、その他の特定のコンテキスト XDM オブジェクトに対して、true または false と評価される決定ルールへの参照。このルールは、そのオプションが特定のプロファイルに該当するかどうかを判断するために使用します。値は、参照される決定ルールの URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/rule を参照してください。
   **型：**&#x200B;文字列

* **_エクスペリエンス/判定/条件/profileConstraints/プロファイル制約タイプ**

   **フィールド：** profileConstraintType
   **タイトル：**プロファイルの制約タイプ
   **説明：**現在制約が設定されているかどうか、および制約の表現方法を判定します。ルールを通じて、または 1 つ以上のセグメントメンバーシップを通じて決定されます。
   **型：**文字列
   **使用可能な値：**
   * 「なし」 (default)
   * 「eligibilityRule」：「プロファイルの制約は単一のルールとして表され、ルールが true と評価されると、制約付きアクションが許可されます。」
   * 「anySegments」：「プロファイルの制約は 1 つまたは複数のセグメントとして表されます。また、制約付きアクションを許可するには、プロファイルがそれらの少なくとも 1 つのセグメントのメンバーである必要があります。」
   * 「allSegments」：「プロファイルの制約は 1 つまたは複数のセグメントとして表されます。また、制約付きアクションを許可する前に、プロファイルがそれらのすべてのセグメントのメンバーである必要があります。」
   * 「rules」：「プロファイルの制約はさまざまなルール（実施要件、適用性、適合性など）で表され、すべてのルールが true と評価された場合に、制約付きアクションが許可されます。」

**エクスペリエンス/判定/条件/ランキング**

**フィールド：** ranking 
**タイトル：**ランキングの詳細
**説明：**&#x200B;ランク（優先度）。決定条件のコンテキストに基づいて「最良のオプション」と判定する方法を定義します。プロファイルの制約を満たす、選択したすべてのオプションの中から、ランキングに基づいて最良の（または上位 N 個の）提案オプションを決定します。**型：**&#x200B;オブジェクト

* **エクスペリエンス/判定/条件/ランク/順序**

   **フィールド：** order
   **タイトル：**順序の評価
   **説明：**1 つ以上の決定オプションの相対的な順序の評価。順序の値が大きいオプションが、順序の値が小さいオプションよりも優先されます。このメソッドで決められる値は、順序としての意味を持ちますが、差を測定したり、和や製品を計算することはできません。主な傾向を測る数値として序数データに使用できるのは、中央値とモードだけです。
   **型：**&#x200B;オブジェクト

   * **スコアリング関数**

      **フィールド：** function
      **タイトル：**スコアリング関数
      **説明：**この決定オプションの数値スコアを計算する関数への参照。このスコアに基づいて決定オプションの順序（ランク）が決まります。このプロパティの値は、on オプションで一度に呼び出す関数の URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/function を参照してください。
      **型：**&#x200B;文字列

   * **順序評価タイプ**

      **フィールド：** orderEvaluationType
      **タイトル：**順序評価タイプ
      **説明：**使用する順序評価メカニズム、決定オプションの静的優先度、すべてのオプションの数値を計算するスコアリング関数、リストを受け取って順序を指定するランキング戦略を指定します。
      **型：**文字列
      **使用可能な値：**「static」、「scoringFunction」、「rankingStrategy」

   * **ランキング戦略**

      **フィールド：** rankingStrategy
      **タイトル：**ランキング戦略
      **説明：**決定オプションリストのランク付けをおこなう戦略への参照。決定オプションは、順序付けされたリストで返されます。このプロパティの値は、on オプションで一度に呼び出す関数の URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/rankingStrategy を参照してください。
      **型：**&#x200B;文字列

* **_エクスペリエンス/判定/条件/ランク/優先度**

   **フィールド：** priority
   **タイトル：**優先度
   **説明：**1 つの決定オプションの優先度を、その他のすべてのオプションと比較した場合の優先度。順序関数が指定されていないオプションは、このプロパティを使用して優先付けされます。優先度の高いオプションは、優先度の低いオプションよりも先に選択されます。条件を満たす、優先度の値が最高のオプションが複数ある場合、1 つがランダムに選択され、決定の提案に使用されます。
   **型：**整数
   **最小値：** 0
   **デフォルト値：** 0

**型：**&#x200B;文字列
