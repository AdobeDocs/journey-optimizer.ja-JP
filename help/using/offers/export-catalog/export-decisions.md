# 決定データセット{#decisions-dataset}

オファーが変更されるたびに、判定用の自動生成データセット（旧称：アクティビティ）が更新されます。

![](../../assets/dataset-activities.png)

正常に更新された最新のデータセットのバッチが右側に表示されます。 データセットのスキーマの階層表示が左側のペインに表示されます。

>[!NOTE]
>
>オファーライブラリの各オブジェクト用にエクスポートしたデータセットにアクセスする方法については、[このセクション](../export-catalog/access-dataset.md)を参照してください。

以下に、**[!UICONTROL 決定オブジェクトリポジトリ - 決定]**（旧称は決定オブジェクトリポジトリ - アクティビティ）データセットで使用できるすべてのフィールドのリストを示します。

**_エクスペリエンス/判定/条件/配置**

**フィールド：** 配置
**タイトル：** 配置制限
**説明：** 配置制約は、この条件がリストに表示されている配置にのみ適用できることを示します。対象となるプレースメントが`xdm:placements`リスト内にある場合にのみ、オプションの選択が考慮されます。それ以外の場合は、決定条件全体がスキップされます。 &#39;xdm:placements&#39; リストが省略されたり空の場合、対象の任意のプレースメントに対して条件が考慮されます。 ここに示すプレースメントは、オプション選択に対して暗黙の条件を課します。 考慮するオプションには、対象となるプレースメントの表現が必要です。**型：**&#x200B;配列

* **プレースメント識別情報**

   **タイトル：**プレースメント識別情報
   **説明：**プレースメントエンティティへの参照。値は、参照されるプレースメントの URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/placement を参照してください。
   **型：**&#x200B;文字列

**_エクスペリエンス/判定/条件/profileConstraints**

**フィールド：** profileConstraints
**タイトル：** プロファイル制約説
**明：** プロファイル制約は、このコンテキストにおいて、オプション選択が現時点でこのプロファイルIDに対して適格かどうかを決定します。プロファイル制約で各オプションの値を考慮する必要がない場合、つまり、オプションの選択肢が不変である場合、プロファイル制約が「false」と評価されると、オプション選択全体がキャンセルされます。一方、オプションをパラメーターとするプロファイル制約ルールは、オプションの選択で適合する各オプションに対して評価されます。
**型：**&#x200B;オブジェクト

* **_エクスペリエンス/判定/条件/profileConstraints/説明**

   **フィールド：** 説明
   **タイトル：**説明
   **説明：**プロファイル制約の説明。このプロファイル制約がなぜ、どのように設けられたか、含まれるオプションや除外されるオプションは何かについて、人間が理解できる形で意図を伝えるために使用します。
   **型：**&#x200B;文字列

* **_エクスペリエンス/判定/条件/ profileConstraints /実施要件ルール**

   **フィールド：**eligibilityRule
   **タイトル：**実施要件ルール
   **説明：**特定のプロファイルやその他の指定されたコンテキスト XDM オブジェクトに対して、true または false と評価される決定ルールへの参照。このルールは、そのオプションが特定のプロファイルに該当するかどうかを判断するために使用します。 値は、参照される決定ルールの URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/rule を参照してください。
   **型：**&#x200B;文字列

* **_エクスペリエンス/判定/条件/profileConstraints/プロファイル制約タイプ**

   **フィールド：**profileConstraintType
   **タイトル：**プロファイル制約タイプ
   **説明：**現在制約が設定されているかどうか、および制約が明確であるかを判定します。ルールを通じて、または 1 つ以上のセグメントメンバーシップを通じて決定されます。
   **型：**文字列
   **取り得る値：**
   * 「なし」 (default)
   * 「eligibilityRule」：プロファイル制約は単一のルールとして表され、ルールが true と評価された場合に、制約付きアクションが許可されます。
   * 「anySegments」：プロファイル制約は 1 つまたは複数のセグメントとして表され、プロファイルがそれらの少なくとも 1 つのセグメントのメンバである場合に、制約付きアクションが許可されます。
   * 「allSegments」：プロファイル制約は 1 つまたは複数のセグメントとして表され、プロファイルがそれらのすべてのセグメントのメンバである場合に、制約付きアクションを許可されます。
   * 「rules」：プロファイル制約はさまざまなルール（実施要件、適合性、適切さなど）で表され、すべてのルールが true と評価された場合に、制約付きアクションが許可されます。

**エクスペリエンス/判定/条件/ランキング**

**フィールド：**ranking 
**タイトル：**ランキング詳細
**説明：**&#x200B;ランク（優先度）。決定条件のコンテキストに基づいて「最良のオプション」と判定する方法を定義します。 プロファイル制約を満たす、選択したすべてのオプションの中から、ランキングに基づいて最良の（または上位 N 個の）提案オプションを決定します。**型：**&#x200B;オブジェクト

* **エクスペリエンス/判定/条件/ランク/順序**

   **フィールド：**order
   **タイトル：**順序評価
   **説明：**1 つまたは複数の決定オプションの相対的な順序の評価。順序の値が大きいオプションが、順序の値が小さいオプションよりも優先されます。 このメソッドで決められる値は、順序としての意味を持ちますが、差を計算したり、和や積を計算することはできません。主な傾向を測る数値として序数データに使用できるのは、中央値とモードだけです。
   **型：**&#x200B;オブジェクト

   * **スコアリング関数**

      **フィールド：**function
      **タイトル：**スコアリング関数
      **説明：**この決定オプションの数値スコアを計算する関数への参照。このスコアに基づいて決定オプションの順序（ランク）が決まります。このプロパティの値は、on オプションで一度に呼び出す関数の URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/function を参照してください。
      **型：**&#x200B;文字列

   * **順序評価タイプ**

      **フィールド：**orderEvaluationType
      **タイトル：**順序評価タイプ
      **説明：**使用する順序評価メカニズムを指定します。評価メカニズムには、決定オプションの静的優先度、すべてのオプションの数値を計算するスコアリング関数、リストを受け取ってオプションの順序を決めるランキング戦略があります。
      **型：**文字列
      **取り得る値：**「static」、「scoringFunction」、「rankingStrategy」

   * **ランキング戦略**

      **フィールド：**rankingStrategy
      **タイトル：**ランキング戦略
      **説明：**決定リストのランク付けを行う戦略の参照。決定オプションは、順序付けされたリストで返されます。 このプロパティの値は、on オプションで一度に呼び出す関数の URI（@id）です。スキーマ https://ns.adobe.com/experience/decisioning/rankingStrategy を参照してください。
      **型：**&#x200B;文字列

* **_エクスペリエンス/判定/条件/ランク/優先度**

   **フィールド：**priority
   **タイトル：**優先度
   **説明：**1 つの決定オプションの優先度を、その他のすべてのオプションと比較した場合の優先度。順序関数が指定されていないオプションは、このプロパティを使用して優先付けされます。 優先度の高いオプションは、優先度の低いオプションよりも先に選択されます。 条件を満たす、優先度が最高のオプションが複数ある場合、片方が一様にランダムに選択され、決定の提案に使用されます。
   **型：**整数
   **最小値：**0
   **デフォルト値：** 0
