# メールへの CC フィールドの追加 {#cc-email-field}

>[!CONTEXTUALHELP]
>id="ajo_admin_config_cc"
>title="CC メールアドレスの定義"
>abstract="このチャネル設定で送信されるメールに、表示可能な CC （カーボンコピー）フィールドを追加できます。 固定メールアドレスを入力するか、パーソナライゼーション（プロファイル属性またはコンテキスト変数）を使用します。 CC の使用は、資格のあるメッセージ量にカウントされることに注意してください。"

>[!AVAILABILITY]
>
>この機能は、限定提供（LA）のすべてのお客様が利用できます。 アクセス権を取得するには、アドビ担当者にお問い合わせください。

ジャーニーとキャンペーンを通じて [!DNL Journey Optimizer] ーザーから送信されるメールに、表示可能な CC （カーボンコピー）フィールドを追加できます。 このオプション機能は、メールヘッダーパラメーターおよび「BCC メール」オプションと共に、[&#x200B; チャネル設定 &#x200B;](channel-surfaces.md) レベルで設定されます。

>[!CAUTION]
>
>CC 機能の使用状況は、ライセンスを取得したメッセージ数に照らしてカウントされます。 表示可能な CC 受信者が必要な場合にのみ有効にします。 ライセンスを取得したボリュームを契約書で確認します。

[BCC](archiving-support.md#bcc-email) と同様に、「CC」フィールドは、メールのコピーを追加のアドレスに送信することを目的としています。 ただし、BCC とは次の点で異なります。

* CC メールアドレスはプライマリ受信者に表示されるので、プライマリ受信者はコピーされたユーザーを確認でき、フォローアップのために誰に連絡すればよいかを把握できます。
* BCC とは異なり、「CC メール」フィールドは、パーソナライゼーションをサポートし、多くのシナリオで 1 つのチャネル設定を使用し、受信者（関係マネージャーなど）ごとに適切な人物にコピーを送信できます。 API トリガーキャンペーンの場合は、特定のイベントまたはトランザクションに関連するアドレスを CC で送信できます。

>[!NOTE]
>
>アーカイブやコンプライアンスのために受信者にアドレスを表示できない場所にコピーを保存する必要がある場合は、CC ではなく [BCC](archiving-support.md#bcc-email) を使用します。

## CC メールを有効にする {#enable-cc}

「**[!UICONTROL CC メール]**」オプションを有効にするには、[&#x200B; メールチャネル設定 &#x200B;](../email/email-settings.md) の「CC」フィールドを設定します。

![](assets/email-config-cc.png)

アドビにデリゲートしたサブドメインで定義されたメールアドレスを除き、正しい形式で任意の外部アドレスを指定できます。例えば、*marketing.luma.com* サブドメインをアドビにデリゲートした場合、*abc@marketing.luma.com* のようなアドレスは禁止されます。

>[!CAUTION]
>
>1 つのメールアドレスのみを定義できます。 現在のチャネル設定を使用して送信されるすべてのメールを保存するのに十分な受信容量が CC アドレスにあることを確認します。
>
>その他のレコメンデーションの一覧について詳しくは、[この節](#cc-recommendations-limitations)を参照してください。

「**[!UICONTROL CC メール]**」フィールドには、次の 3 種類の値を入力できます。

* **ハードコーディングされた値**。固定のメールアドレスにすることができます。

* プロファイルで使用可能な関係マネージャーのメールアドレスなどの **プロファイル属性**。

* **コンテキスト属性** – この値は **API トリガーキャンペーンでのみ使用** できます。 API ペイロードから取得されます。このペイロードには、CC アドレス値を持つコンテキスト変数 `context.channel.email.ccvalues` を含める必要があります。

  >[!WARNING]
  >
  >**コンテキスト変数** を使用して CC を設定した場合、**API トリガーキャンペーン** でのみサポートされます。 ジャーニーまたはアクションキャンペーンでそのチャネル設定を使用する場合、メールはプライマリ受信者にのみ送信され、CC アドレスにはコピーが送信されません。

メッセージに含まれているすべての [&#x200B; 添付ファイル &#x200B;](../email/pdf-attachments.md) は、プライマリ受信者と CC アドレスの両方に送信されます。

送信時に CC 値が無効または見つからない場合（空のコンテキスト変数など）、CC コピーはスキップされます。プライマリ受信者は引き続きメールを受信します。

「CC」フィールドに複数の値がある場合（例えば、複数のアドレスに解決されるプロファイル属性や式を使用する場合）、最初の値のみがメールの送信に使用されます。

## 既存のチャネル設定で CC メールを編集 {#cc-edit}

[&#x200B; メール設定を編集 &#x200B;](channel-surfaces.md#edit-channel-surface) して「CC」フィールドを追加または変更した場合は、以下を使用することのみ可能です。

* **ハードコード** CC メールアドレス、または
* **コンテキスト変数** （API トリガーでの使用用）。

>[!CAUTION]
>
>既存のメールチャネル設定を編集する場合、新しい [&#x200B; プロファイル属性 &#x200B;](../personalization/personalization-build-expressions.md#sources) を「**[!UICONTROL CC メール]**」フィールドに追加できません。 [&#x200B; 新規チャネル設定 &#x200B;](channel-surfaces.md#create-channel-surface) を作成する必要があります。

## レコメンデーションと制限事項 {#cc-recommendations-limitations}

* **使用権限：** CC の使用状況は、資格のあるメッセージボリュームにカウントされます。 表示可能な CC 受信者が必要な場合にのみ、CC を使用します。 ライセンスを取得したボリュームを契約書で確認します。

* **プライバシーとコンプライアンス：** プライバシーコンプライアンスを確保するために、CC メールは、該当する場合、個人情報（PII）を安全に保存できるシステムで処理する必要があります。 メッセージには、PII などの機密データや非公開データを含めることができるので、CC アドレスが正しいことを確認し、メッセージへのアクセスを保護します。

* **インボックス管理：** CC に使用するインボックスは、領域と配信について適切に管理される必要があります。 インボックスがバウンスを返した場合、一部のメールが受信されない場合があります。

* **配信タイミング：** メッセージは、ターゲット受信者より前に、CC メールアドレスに配信される場合があります。 CC メッセージは、元のメッセージが [&#x200B; バウンス &#x200B;](../reports/suppression-list.md#delivery-failures) された場合でも送信できます。

* **レポート：** CC 受信者からの開封数、クリック数などのエンゲージメントは、メールレポート指標に含まれます。 したがって、CC 受信者が開いたりクリックしたりすると、[reports](../reports/report-gs-cja.md) で計算ミスが発生します。

* **スパム：** CC インボックスでメッセージをスパムとしてマークしないでください。このアドレスに送信される他のすべてのメールに影響を与えます。

* **配信品質：** 送信方法と受信者の期待に沿って CC を使用します。 CC の過度の使用は、配信品質、つまり [&#x200B; 配信品質のベストプラクティス &#x200B;](../reports/deliverability.md) や契約条件に影響を与える可能性があります。

>[!CAUTION]
>
>CC アドレスに送信されたメールの購読解除リンクをクリックしないでください。この場合、メールの **宛先** 受信者の登録が直ちに解除されます。
