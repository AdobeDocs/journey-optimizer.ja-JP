# IP ウォームアップ配信品質ガイド {#ip-warmup-deliverability-guide}

Adobe Journey Optimizer で新しい IP アドレスまたはドメインを使用してメールキャンペーンを開始する場合、強力な送信者の評判を確立するために、配信品質の基本を理解することが重要です。このガイドでは、ゼロ評判からインボックスへの配置を正常に移行するのに役立つ主な概念、準備手順、ベストプラクティスについて説明します。

➡️ 配信品質の基本、評判の確立、IP ウォームアップのベストプラクティスについて詳しくは、この[アドビのブログ投稿](https://experienceleaguecommunities.adobe.com/t5/journey-optimizer-blogs/adobe-journey-optimizer-deliverability-guide-from-zero/ba-p/761950){target="_blank"}のビデオを参照してください。

>[!NOTE]
>
>Adobe Journey Optimizer で IP ウォームアッププランを実装する手順について詳しくは、[IP ウォームアッププランの基本を学ぶ](ip-warmup-gs.md)を参照してください。

## IP とドメインの評判が重要な理由 {#reputation-matters}

メールボックスプロバイダー（Gmail、Yahoo、Microsoft、Apple メールなど）は、次の 4 つの主要な柱に基づいてすべての送信者を評価します。

* **苦情**：受信者はメッセージをスパムとしてマークしましたか？
* **エンゲージメント**：受信者はメールを開封、クリックまたは返信しますか？
* **インフラストラクチャ**：送信インフラストラクチャは認証済みで、安定しており、信頼性が高いですか？
* **コンテンツ**：メッセージコンテンツは正当で価値のあるものに見えますか？

IP ウォームアップは、完全な送信ボリュームに拡張する前に、メールボックスプロバイダーに対して新しいインフラストラクチャが正当であり、要求されていることを段階的に示すことにより、主に最初の 3 つの柱に対処します。

## プリフライトのチェックリスト {#pre-flight-checklist}

IP アドレスのウォームアップを開始する前に、すべての基本要素が揃っていることを確認します。次の表に、IP ウォームアッププランを開始する前に完了する必要がある基本的なタスクの概要を示します。

| タスク | これが重要な理由 | 実行方法 |
|------|----------------|-------------------|
| AJO で固定 IP を予約し、サブドメインをデリゲート | すべての今後の評判は、これらのインフラストラクチャ要素に結びつきます。 | **[!UICONTROL 管理]**／**[!UICONTROL チャネル]**／**[!UICONTROL メール設定]**／**[!UICONTROL サブドメイン]**&#x200B;に移動します。[詳細情報](delegate-subdomain.md) |
| SPF とDKIM を設定 | 送信サーバーが正当で承認済みであることを確認します。 | [サブドメインのデリゲーション](delegate-subdomain.md)と[チャネル設定の作成](channel-surfaces.md)後に、アドビにより自動的に処理されます。 |
| DMARC レコードが設定されていることを確認（p=なし） | メール認証レポートと今後の適用ポリシーを有効にします。 | すべてのデリゲートされたサブドメインに対して DMARC レコードが設定されていることを確認します。新しいサブドメインをデリゲートする際は、インターフェースで直接 DMARC を設定できます。[詳細情報](dmarc-record.md) |
| シードリスト監視を設定 | ウォームアッププロセスの早い段階で、インボックスへの配置の問題を検出します。 | チャネル設定を作成する際に、シードアドレスを追加します。[詳細情報](seed-lists.md) |
| フェーズ 1 でエンゲージメントの高いオーディエンスを作成 | 最もアクティブな受信者との初期エンゲージメント指標を向上させます。 | 過去 30 日間に開封またはクリックした 5,000 未満の連絡先のオーディエンスを作成します。[詳細情報](../audience/creating-a-segment-definition.md) |

>[!CAUTION]
>
>認証の問題（SPF、DKIM、DMARC）は、ボリュームの増加では解決できません。送信を開始する前に、これらが正しく設定されていることを確認します。

## 4 週間のウォームアップカレンダーのサンプル {#warmup-calendar}

このサンプルカレンダーでは、最終的な 1 日の量（UDV）の割合に基づいて、プログレッシブなボリュームの増加を提供します。特定の送信要件に合わせてこれらの数値を調整し、配信品質コンサルタントと協力してカスタマイズされたプランを作成します。

| 数日 | UDV の割合 | ターゲットオーディエンス | コンテンツのレコメンデーション |
|------|----------|-----------------|------------------------|
| 1～3 | 0.5％ | 最も関与した受信者 | 短いプレーンテキスト形式を使用し、上部に明確なコールトゥアクションを表示します。 |
| 4～7 | 1％ | 関与したユーザーと最近の購入者 | 軽量のヒーロー画像を追加し、リンクを 3 以下に制限します。 |
| 8～14 | 5％ | より広範なアクティブサブスクライバーリスト | 標準のメールテンプレートを導入しますが、大量のプロモーションコンテンツは回避します |
| 15～21 | 25％ | アクティブなサブスクライバーと、わずかに非アクティブなサブスクライバー | 苦情率を詳細に監視しながら、通常のマーケティングコンテンツを使用します。 |
| 22～28 | 50～100％ | 完全なリスト（抑制リストの適用） | 定常送信頻度に移行します。 |

## IP ウォームアッププラン機能の使用 {#ajo-warmup-feature}

Adobe Journey Optimizer には、効率化された [IP ウォームアッププラン](ip-warmup-gs.md)機能が含まれており、複雑なジャーニー設定を通じて手動のボリュームキャップを行う必要がなくなります。この機能により、送信者の評判を確立するための標準化されたアプローチが確保されます。

### 仕組み

1. **IP ウォームアップキャンペーンを作成**：「**[!UICONTROL IP ウォームアッププランのアクティブ化]**」オプションを有効にしたキャンペーンを 1 つ以上作成します。[詳細情報](ip-warmup-campaign.md)

1. **プランを設定**：**[!UICONTROL 管理]**／**[!UICONTROL チャネル]**／**[!UICONTROL メール設定]**／**[!UICONTROL IP ウォームアッププラン]**&#x200B;にアクセスし、配信品質コンサルタントと準備した段階的なウォームアップテンプレートをアップロードします。[詳細情報](ip-warmup-plan.md)

1. **フェーズを実行**：各フェーズのキャンペーンを選択し、対応する実行をアクティブ化します。過剰な連絡を防ぐために、以前の実行からプロファイルは自動的に除外されます。[詳細情報](ip-warmup-execution.md)

1. **監視と調整**：統合レポートダッシュボードを使用して、進行状況の追跡、実行ステータスの監視、問題が発生した場合のプランの変更を行います。[詳細情報](ip-warmup-execution.md#monitor-plan)

## リアルタイム監視と主要指標 {#monitoring}

Adobe Journey Optimizer には、IP ウォームアップのパフォーマンスを追跡するための次のビルトインレポート機能が用意されています。

* **ライブレポート**：「**[!UICONTROL 過去 24 時間]**」タブからキャンペーンのリアルタイム測定とビジュアライゼーションにアクセスします。[詳細情報](../reports/campaign-live-report.md#email-live)

* **全期間のレポート**：より深いインサイトを得るには、Customer Journey Analytics を活用して Adobe Experience Platform のデータを分析し、カスタムビジュアライゼーションを作成します。[詳細情報](../reports/report-gs-cja.md)

### ターゲット指標

ウォームアップを通じて次の主要業績評価指標を監視します。

| 指標 | ターゲットしきい値 | 是正措置 |
|--------|-----------------|-------------------|
| 苦情率 | 0.1％以下 | 超過した場合は、セグメントを監査し、慢性的に苦情を訴える顧客を抑制します。 |
| ハードバウンス率 | 2％以下 | 超過した場合は、リストの品質とハイジーンの手法をレビューします。 |
| 開封率 | 10％以上 | 低すぎる場合は、関与したオーディエンスをターゲットにしていることを確認します。 |

## トラブルシューティングプレイブック {#troubleshooting}

ウォームアップ中によくある問題に対処するには、次の決定マトリックスを使用します。

| 症状 | 考えられる原因 | 推奨されるアクション |
|---------|--------------|-------------------|
| Yahoo の一時的なエラー（421 エラー） | ボリュームの増加が早すぎる | 24 時間送信を一時停止してから、前の層でやり直します。 |
| シードアカウント全体で開封率が 2％を下回る | IP ブロックリストへの登録 | [Google Postmaster Tools](https://postmaster.google.com/) と [Microsoft SNDS](https://sendersupport.olc.protection.outlook.com/snds/) を確認し、必要に応じて配信品質チケットを開きます。 |
| 苦情率が 0.3％を超えている | ターゲットにミスがあるか、オーディエンスが古い | セグメント定義を監査し、慢性的に苦情を訴える顧客を[抑制リスト](manage-suppression-list.md)から除外します。 |

>[!IMPORTANT]
>
>傷ついた送信者の評判を後で修復しようとするよりも、ウォームアップをゆっくり進める方が効果的です。積極的なボリュームの増加よりも健全な指標の維持を常に優先します。

## ウォームアップ後のベストプラクティス {#post-warmup}

ウォームアッププランを完了し、指標が安定したら、以下の点に留意してください。

* **一貫性を維持**：確立した評判を維持するために、1 日の量の増加率を前週比 30％未満に抑えます。

* **継続的に監視**：四半期ごとに評判のヘルスチェックをスケジュールし、問題をプロアクティブに特定して対処します。

* **エンゲージメントシグナルを適用**：引き続き関与した受信者を優先し、非アクティブなサブスクライバーには再エンゲージメントキャンペーンを実装します。

* **認証を最新の状態に保持**：SPF、DKIM、DMARC レコードが適切に設定されていることを定期的に確認します。

## 重要な留意点 {#key-takeaways}

* **IP ウォームアップが不可欠**：ウォームアッププロセスをスキップすると、適切に行うために必要な労力よりも多くの時間と評判が損なわれます。

* **データ駆動型の決定**：苦情、バウンス、エンゲージメント率を毎日追跡し、ISP からペナルティを受ける前に戦略を調整します。

* **認証を 1 番目に、ボリュームを 2 番目に**：ボリュームの増加を開始する前に、SPF、DKIM、DMARC に関するすべての問題を解決します。

* **一貫性が重要**：メールボックスプロバイダーは予測可能な送信パターンを好みます。突然のボリュームの増加や不規則な送信スケジュールは回避します。
