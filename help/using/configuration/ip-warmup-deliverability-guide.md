# IP ウォームアップ配信品質ガイド {#ip-warmup-deliverability-guide}

Adobe Journey Optimizerで新しい IP アドレスまたはドメインを使用したメールキャンペーンを開始する場合、配信品質の基本事項を理解することは、送信者の強力なレピュテーションを構築するために重要です。 このガイドでは、評判ゼロから成功するインボックスの配置への移行に役立つ、重要な概念、準備手順およびベストプラクティスについて説明します。

➡️ [IP ウォームアップ配信品質の基本については、このビデオをご覧ください &#x200B;](#video)

>[!NOTE]
>
>Adobe Journey Optimizerに IP ウォームアッププランを実装する手順については、[IP ウォームアッププランの基本を学ぶ &#x200B;](ip-warmup-gs.md) を参照してください。

## IP とドメインのレピュテーションが重要な理由 {#reputation-matters}

メールボックスプロバイダー（Gmail、Yahoo、Microsoft、Apple Mail など）は、次の 4 つの主要な柱に基づいてすべての送信者を評価します。

* **苦情**：受信者はメッセージをスパムとマークしましたか？
* **エンゲージメント**：受信者はメールを開いたり、クリックしたり、返信したりしますか？
* **インフラストラクチャ**：送信インフラストラクチャは認証済みで、安定しており、信頼性が高いですか？
* **コンテンツ**：メッセージコンテンツが正当で価値のあるものと思われますか？

IP ウォームアップは、主に、新しいインフラストラクチャが妥当であり、完全な送信ボリュームに拡張する前に必要であることをメールボックスプロバイダーに段階的に示すことで、最初の 3 つの柱に取り組みます。

## 飛行前のチェックリスト {#pre-flight-checklist}

IP アドレスのウォームアップを開始する前に、すべての基本要素が配置されていることを確認します。 次の表に、IP ウォームアッププランを開始する前に完了する必要がある基本的なタスクの概要を示します。

| タスク | これが重要な理由 | を達成する方法 |
|------|----------------|-------------------|
| AJOでの固定 IP の予約とサブドメインのデリゲート | すべての将来的な評価はこれらのインフラストラクチャ要素に添付されます | **[!UICONTROL 管理]**/**[!UICONTROL チャネル]**/**[!UICONTROL メール設定]**/**[!UICONTROL サブドメイン]** に移動します。 [詳細情報](delegate-subdomain.md) |
| SPF とDKIMの設定 | 送信サーバーが正規のものであり、承認されていることを確認します | サブドメインのデリゲーションおよびチャネル設定の作成後に、Adobeによって自動的に処理されます。 [詳細情報](delegate-subdomain.md) |
| DMARC レコードの設定 | 電子メール認証レポートおよび今後の適用ポリシーを有効にします | サブドメインのデリゲーションおよびチャネル設定の作成後に、Adobeによって自動的に処理されます。 [詳細情報](dmarc-record.md) |
| シードリスト監視の設定 | ウォームアッププロセスの早期にインボックスの配置の問題を検出 | チャネル設定を作成する際に、シードアドレスを追加します。 [詳細情報](seed-lists.md) |
| エンゲージメントの高いオーディエンスのフェーズ 1 を作成 | 最もアクティブな受信者に対する早期エンゲージメント指標を向上させます | 過去 30 日間に開封またはクリックした連絡先が 5,000 人未満のオーディエンスを作成します |

>[!CAUTION]
>
>認証の問題（SPF、DKIM、DMARC）は、ボリュームランピングでは解決できません。 送信を開始する前に、これらが正しく設定されていることを確認してください。

## 4 週間のウォームアップカレンダーのサンプル {#warmup-calendar}

このサンプルカレンダーでは、最終的な 1 日のボリューム（UDV）の割合に基づいて、プログレッシブボリュームランプを提供します。 特定の送信要件に合わせてこれらの数値を調整し、配信品質コンサルタントと協力してカスタマイズされたプランを作成します。

| 数日 | UDV の % | ターゲットオーディエンス | コンテンツの推奨事項 |
|------|----------|-----------------|------------------------|
| 1-3 | 0.5% | 最もエンゲージメントの高い受信者 | 折りたたみの上に明確なcall-to-actionを含む、短いプレーンテキスト形式を使用します |
| 4-7 | 1% | エンゲージメントの高いユーザーと最近の購入者 | 軽量のヒーロー画像を追加し、リンクを 3 以下に制限します |
| 8-14 | 5% | 広範なアクティブ購読者リスト | 標準のメールテンプレートを紹介しますが、大量のプロモーションコンテンツは避けます。 |
| 15-21 | 25% | アクティブな購読者に、わずかに非アクティブな購読者を加える | 苦情率を詳細に監視しながら、通常のマーケティングコンテンツを使用 |
| 22-28 | 50 ～ 100% | 完全なリスト（抑制リストの順守） | 定常送信頻度への移行 |

>[!NOTE]
>
>Adobe Journey Optimizerには、ボリューム管理を自動化し、複雑なジャーニー設定を必要とせずにウォームアッププロセスを簡素化する、専用の [IP ウォームアッププラン機能 &#x200B;](ip-warmup-gs.md) が用意されています。

## AJOの IP ウォームアッププラン機能の使用 {#ajo-warmup-feature}

Adobe Journey Optimizerには、合理化された IP ウォームアッププラン機能が備わっており、複雑なジャーニー設定を通じて手動でボリュームキャッピングを行う必要がなくなります。 この機能により、送信者のレピュテーションを構築するための標準化されたアプローチが保証されます。

### 仕組み

1. **IP ウォームアップキャンペーンの作成**:「IP ウォームアッププランのアクティブ化 **[!UICONTROL オプションが有効になっているキャンペーンを 1 つ以上作成]** ます。 [詳細情報](ip-warmup-campaign.md)

1. **プランの設定**:**[!UICONTROL 管理]**/**[!UICONTROL チャネル]**/**[!UICONTROL メール設定]**/**[!UICONTROL IP ウォームアッププランにアクセスし]** 配信品質コンサルタントで準備した段階的なウォームアップテンプレートをアップロードします。 [詳細情報](ip-warmup-plan.md)

1. **フェーズを実行**：各フェーズのキャンペーンを選択し、対応する実行をアクティブ化します。 システムは、過剰連絡を防ぐために、以前の実行からプロファイルを自動的に除外します。 [詳細情報](ip-warmup-execution.md)

1. **監視と調整**：統合レポートダッシュボードを使用して、進行状況の追跡、実行ステータスの監視、問題が発生した場合のプランの変更を行います。 [詳細情報](ip-warmup-execution.md#monitor-plan)

## リアルタイムの監視と主要指標 {#monitoring}

Adobe Journey Optimizerには、IP ウォームアップパフォーマンスを追跡するための組み込みのレポート機能が用意されています。

* **ライブレポート**:「過去 24 時間 **[!UICONTROL タブからリアルタイムの測定とキャンペーンのビジュアライゼーションにアクセス]** ます。 [詳細情報](../reports/live-report.md)

* **Customer Journey Analyticsの統合**：より深いインサイトを得るには、Customer Journey Analyticsを活用してAdobe Experience Platformのデータを分析し、カスタムビジュアライゼーションを作成します。 [詳細情報](../reports/report-gs-cja.md)

### ターゲット指標

ウォームアップ全体を通して次の主要業績評価指標を監視します。

| 指標 | ターゲットしきい値 | 超過した場合のアクション |
|--------|-----------------|-------------------|
| 苦情率 | ≤ 0.1% | セグメントの監査と慢性的な苦情の抑制 |
| ハードバウンス率 | ≤ 2% | リストの品質とハイジーンの手法のレビュー |
| 開封率 | ≥ 10% | エンゲージメントのあるオーディエンスをターゲティングしていることを確認 |

>[!TIP]
>
>キャンペーンを包括的に分析するには、{campaign ライブレポート [&#x200B; と &#x200B;](../reports/campaign-live-report.md#email-live)2}Customer Journey Analytics レポート [&#x200B; 機能を使用します。](../reports/campaign-global-report-cja-email.md)

## トラブルシューティングプレイブック {#troubleshooting}

この決定マトリックスを使用して、ウォームアップ中の一般的な問題に対処します。

| 症状 | 考えられる原因 | 推奨されるアクション |
|---------|--------------|-------------------|
| Yahoo 一時的エラー（421 エラー） | ボリュームの増加が早すぎる | 24 時間送信を一時停止した後、前の層で再起動します |
| シードアカウント全体で開封率が 2% を下回る | IPブロックリストへの登録 | [Google Postmaster Tools](https://postmaster.google.com/) および [Microsoft SNDS](https://sendersupport.olc.protection.outlook.com/snds/) を確認します。必要に応じて配信品質チケットを開きます。 |
| 苦情率が 0.3% を超えています | ターゲットが誤っている、またはオーディエンスが古い | セグメント定義を監査し、[&#x200B; 抑制リスト &#x200B;](manage-suppression-list.md) から慢性的な苦情を除外します |

>[!IMPORTANT]
>
>破損した送信者の評判を後で修復するよりも、ウォームアップの速度を下げる方が効果的です。 積極的なボリュームランピングよりも、常に健全な指標の維持を優先します。

## ウォームアップ後のベストプラクティス {#post-warmup}

ウォームアップ計画を完了し、指標が安定したら、次の手順を実行します。

* **一貫性の維持**：確立されたレピュテーションを維持するために、1 日あたりのボリューム増加を週あたり 30% 未満に抑えます

* **継続的な監視**：四半期ごとの評判ヘルスチェックをスケジュールして、問題をプロアクティブに特定し対処します

* **エンゲージメントシグナルの尊重**：エンゲージメントのある受信者の優先順位付けを継続し、非アクティブな購読者に対して再エンゲージメントキャンペーンを実施します

* **認証を最新の状態に保つ**:SPF、DKIM、DMARC レコードが適切に設定されていることを定期的に確認します

## 重要ポイント {#key-takeaways}

* **IP ウォームアップは必須です**：ウォームアッププロセスをスキップすると、適切に行うために必要な労力よりも多くの時間と評判が必要になります

* **データに基づく決定**：苦情、バウンス、エンゲージメントの割合を毎日追跡し、ISP がペナルティを科す前に戦略を調整する

* **認証第一、ボリューム第二**：ボリュームの立ち上げを開始する前に、SPF、DKIM、DMARCのすべての問題を解決します

* **一貫性に関する問題**：メールボックスプロバイダーは、予測可能な送信パターンを採用し、突然のボリュームスパイクや不規則な送信スケジュールを回避します
