# 役割と責務

Adobe Journey Optimizerを使用すると、ブランドはカスタマージャーニーを通じて、コンテキストに応じた、つながりのあるパーソナライズされたエクスペリエンスを提供できます。 Journey Optimizerは、規模、速度、柔軟性に重点を置いたエンドツーエンドで構築されており、次の 3 つの主な価値推進要因を 1 つの統合アプリケーションに統合します。

* Adobeのリアルタイム顧客プロファイルを活用した **リアルタイムの顧客インサイトとエンゲージメント**
* **最新のオムニチャネルオーケストレーション** リアルタイムのジャーニーとバッチキャンペーンの両方に対応する統合キャンバスと、最新のメッセージデザイナーを使用
* 意思決定管理および AI/ML 機能による **インテリジェントな意思決定とパーソナライゼーション**

Journey Optimizerには、様々なマーケティングニーズを満たす 2 つのオーケストレーション方法があります。

* **ジャーニー**：行動やイベントによってトリガーされ、各顧客が自分のペースで進む、リアルタイムの 1 対 1 のエンゲージメントに最適です
* **調整されたキャンペーン**:1 対多のバッチキャンペーンに最適です。オーディエンスは、複数の手順からなるワークフローを通じてスケジュールに従って進捗します。季節的なプロモーション、製品の発売、アカウントベースの通信に最適です

この統一されたエクスペリエンスにより、オーディエンスの定義やジャーニーのデザインから、パーソナライズされたコンテンツの作成や結果の分析に至るまで、ユースケース全体を 1 か所で実装できます。 このドキュメントでは、Journey Optimizer を効果的に使用する際の重要な役割と責任および使用の開始方法について説明します。

**重要なメモ：** Adobe Journey Optimizer では、特定の責任を持つ個別のロールを定義します。組織の構造に応じて、1 人のユーザーが複数の役割またはすべての役割を実行することができます。

## 役割に基づいたクイックスタートガイド

実装を簡単にするために、Adobe Journey Optimizerでは、専門知識に基づいてタスクを固有の役割に整理しています。 各役割は、シームレスなカスタマーエクスペリエンスを提供するのに必要な基本的なタスクに焦点を当てています。

| 役割 | 主要な責任 | 重要なスキル | 典型的なタスク |
|-------------------|----------------------------------|--------------------------------|-----------------------------------------------|
| **管理者** | 環境のセットアップとアクセス管理 | システム設定、ユーザー管理、セキュリティ | サンドボックスの設定、ユーザー権限の管理、チャネルとメッセージプリセットの設定 |
| **データエンジニア** | 顧客プロファイルデータとデータソース | データモデリング、XDM スキーマ、ソースコネクタ | プロファイルとビジネスデータをスキーマにモデル化し、ソースコネクタを設定して、データ取得を監視します |
| **開発者** | 技術的な実装と統合 | モバイル/Web SDK、API、イベント駆動型アーキテクチャ | SDK の統合、イベントの実装、カスタムアクションエンドポイントの作成 |
| **マーケター** | ジャーニーデザインとパーソナライズされたエクスペリエンス | ジャーニーオーケストレーション、コンテンツ作成、オーディエンスターゲット設定 | カスタマージャーニーの設計、メッセージの作成とパーソナライズ、オファーと決定コンポーネントの管理、オーディエンスの定義 |

各役割は、Adobe Journey Optimizerの実装の特定のフェーズに対応し、構造化された効率的なデプロイメントプロセスを保証します。

## 実装順序と役割の依存関係

Journey Optimizer の実装が成功すると、通常、このシーケンスに従います。このシーケンスは、役割間の依存関係を反映したものです。

1. **管理者**：環境を設定します\
   管理者は、サンドボックスの設定、アクセス制御の設定、チャネル設定の準備を行うことで、基盤を確立します。 これは、他のチームが機能できるようにするために、最初に行う必要があります。
   * 開発、ステージングおよび実稼動サンドボックスの設定
   * 役割、権限、オブジェクトレベルのアクセス制御（OLAC）の設定
   * チャネル設定（メール、SMS、プッシュ、アプリ内、web、コンテンツカード）を設定
   * サブドメインのデリゲートと IP プールの設定
   * 抑制リストと同意ポリシーの設定

2. **データエンジニア**：データ基盤を作成します。\
   データエンジニアは、パーソナライゼーションを強化するデータインフラストラクチャを構築し、顧客データがシステムに出入りする方法を定義します。
   * 顧客 ID 用の ID 名前空間の作成
   * XDM スキーマ（プロファイル、エクスペリエンスイベント、リレーショナル）の設計
   * データセットを設定して、リアルタイム顧客プロファイルに対して有効にします
   * データ取り込みの設定（バッチおよびストリーミング）
   * 複雑な計算のための計算属性の作成
   * ジャーニーのイベントとデータソースの設定

3. **開発者**：技術統合を実装します\
   開発者は、SDK の統合、イベントの送信、API エンドポイントの構築を行うことで、アプリケーションをJourney Optimizerに結び付けます。 これらの実装により、ジャーニーはトリガーと実行が可能になります。
   * Mobile SDK（iOS/Android）とプッシュ通知設定の統合
   * Web エクスペリエンス用の Web SDKの実装
   * アプリケーションからトリガージャーニーへのイベントの送信
   * 外部システム統合用のカスタムアクションエンドポイントの作成
   * Adobe Experience Platform Assuranceを使用した実装のテスト

4. **マーケター**：顧客体験をデザインし実行する\
   マーケターは、あらゆる基本作業を活用して、すべてのチャネルで、ジャーニーを構築し、コンテンツを作成し、顧客体験を最適化します。
   * セグメント化、CSV アップロードまたはオーディエンスコンポジションを使用したオーディエンスの作成
   * AI アシスタントとテンプレートでパーソナライズされたコンテンツをデザイン
   * イベントとオーディエンスのトリガーを使用したマルチチャネルジャーニーの作成
   * ローンチ前に承認ワークフローでテスト
   * パフォーマンスの監視と、レポートインサイトに基づく最適化

**メモ：**&#x200B;このシーケンスは一般的なものですが、一部のアクティビティが並行して実行されることがあります。例えば、開発者がアプリ統合を行っているときに、データエンジニアがスキーマを設定できます。

## はじめに（役割別）

各役割は、焦点に合わせて調整された特定のタスクから開始されます。これらの最初の手順を完了することで、オンボーディングがスムーズになり、全体的な実装プロセスとの整合性が確保されます。

### マーケター向け {#for-marketers}

マーケターまたはビジネス実務担当者は、すべてのタッチポイントにわたって個人的でコンテキストに沿ったエクスペリエンスを提供するカスタマージャーニーを設計します。 統合されたインターフェイスで作業して、最初から最後までユースケース全体を実装します。

**使用する主な機能：**

* **Journey Orchestration**：様々なチャネルをまたいだ行動やイベントをトリガーとして、各ユーザーが自分のペースで顧客エンゲージメントを進めていく、1 対 1 のリアルタイムのカスタマーエンゲージメントを作成します
* **キャンペーンオーケストレーション**：視覚的なキャンバスを使用して、複雑な複数手順のバッチキャンペーンを大規模に設計および自動化します。 季節的なプロモーション、製品の発売、アカウントベースのコミュニケーションなど、ブランドが開始したキャンペーンに最適です。 マルチエンティティのセグメント化を活用して、顧客データを関連するエンティティ（アカウント、購入、予約）と接続することで、正確なオーディエンスを作成します
* **最新のメッセージDesigner**：ドラッグ&amp;ドロップのインターフェイスを使用して、メールおよびモバイルメッセージをデザインしてパーソナライズします。 標準のテンプレートを編集して、市場投入までの時間を短縮します。
* **意思決定管理**：メールおよび顧客タッチポイントに埋め込むことができる一元化されたライブラリで、オファー、実施要件ルールおよびその他のコンポーネントを作成および管理します
* **アセット管理**:Journey Optimizerに完全に組み込まれた Adobe Experience Manager Assets Essentials にアクセスして、アセットへのアクセスと配信を効率化します
* **オーディエンス定義**：リレーショナルクエリを使用して即座に絞り込み、正確なオーディエンス数を実現する送信前表示を使用して、オンデマンドオーディエンスを構築します
* **AI/ML サービス**：送信時間の最適化と予測エンゲージメントスコアを活用して、価値の高い顧客をターゲットにし、チャーンリスクを最小限に抑えます

**開始：** ユースケーステンプレートとウィザードを使用して、新しいカスタマージャーニーを簡単に作成およびデプロイします。

[マーケター→ーザー向けの基本を学ぶ](path/marketer.md)

### データエンジニア向け {#for-data-engineers}

データアーキテクトまたはエンジニアは、顧客プロファイルデータや、Journey Optimizerで調整されたエクスペリエンスを強化するその他のデータソースを設定および維持管理します。

**主な責務：**

* **顧客プロファイルデータ**：顧客プロファイルデータとビジネスデータをスキーマにモデル化して、顧客の統合された 360 度のビューを作成します
* **リレーショナルデータモデリング**：調整されたキャンペーンの場合は、複数エンティティのセグメント化を可能にするリレーショナルスキーマを設計して、顧客データを関連エンティティ（アカウント、購入、購読、予約など）に接続し、柔軟にオーディエンスを作成できるようにします
* **Source コネクタ**: ソースコネクタを設定して、web、CRM、オフラインデータ、その他のソースからAdobe Experience Platformにデータを取り込みます
* **ID 解決**:ID 名前空間を設定して、リアルタイムでプロファイルを継続的に更新し、セグメントやジャーニーに顧客を出入りさせます
* **データソース**：カスタマージャーニー全体を通じて外部シグナルをリアルタイムでリッスンするようにデータソースを設定します
* **プロファイル管理**：リアルタイム顧客プロファイルのデータセットを有効にして、パーソナライズされたエクスペリエンスを強化します
* **データ品質**：データ取得を監視して、すべてがJourney Optimizerにスムーズに送られるようにします

**開始：** 最初の顧客プロファイルスキーマをモデル化し、ソースコネクタを設定してデータの取り込みを開始します。

[データエンジニア向けの基本を学ぶ→](path/data-engineer.md)

### 管理者向け {#for-administrators}

管理者は、チームが効率的かつ安全に作業できるようにJourney Optimizer環境を設定します。

**主な責務：**

* **サンドボックス**：サンドボックスを作成および管理して、様々なユーザーグループ（開発、テスト、実稼動）のデータとジャーニーをパーティション化します
* **User Management**：ユーザーグループと権限を設定して、様々な機能へのアクセスを制御します
* **チャネル設定**：配信チャネルとメッセージプリセットを設定して、Journey Optimizerを通じて配信されるメッセージとアセット全体で一貫したブランディングを確保します
* **セキュリティとガバナンス**：オブジェクトレベルのアクセス制御（OLAC）の適用、同意ポリシーの設定、データガバナンスポリシーの実装をおこないます
* **配信品質**：サブドメインのデリゲート、IP プールの作成、抑制リストと許可リストの管理をおこないます
* **ジャーニーの設定**: チームのジャーニー要素と設定を設定します

**開始：** サンドボックスとユーザー権限を設定してから、最初のチャネル設定とメッセージプリセットを設定します。

[管理者向けの基本を学ぶ→](path/administrator.md)

### 開発者向け {#for-developers}

Journey Optimizerをアプリケーションに接続する技術統合を実装する。

**主な責務：**

* Adobe Experience Platform Mobile SDKの統合（iOS/Android）
* Web エクスペリエンスおよび web プッシュ通知用の web SDKの実装
* プッシュ通知の資格情報と証明書を設定
* アプリケーションからトリガージャーニーへのイベントの送信
* カスタムアクションを使用してJourney Optimizerが呼び出す API エンドポイントのビルド
* Web、モバイルおよびその他のサーフェス用のコードベースのエクスペリエンスを実装する
* Adobe Experience Platform Assuranceを使用した実装のテストとデバッグ
* Journey Optimizer API を使用したプログラムによるアクセス

**開始：** モバイルまたは web SDKを統合してから、最初のイベントを実装してジャーニーをトリガーします。

[ 開発者→ーザー向けの基本を学ぶ ](path/developer.md)

## Cross-Role Collaboration

Journey Optimizerを正常に実装するには、次のようなすべての役割でのコラボレーションが必要です。

* **管理者** サンドボックス、権限、チャネル設定を設定して、他の役割を有効にします
* **データエンジニア** は、開発者とマーケターが構築するデータ基盤を提供します
* **開発者** マーケターがジャーニーの開発に使用する技術的トリガーを実装します
* **マーケター** データ品質、機能リクエスト、ユーザーエクスペリエンスに関するフィードバックをすべてのチームに提供します

**ベストプラクティス：** 部門横断的な定期的なミーティングを開催して、優先度に合わせて調整し、進捗を共有し、チーム間でブロッカーに対処します。
