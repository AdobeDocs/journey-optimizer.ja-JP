# 役割と責務

Adobe Journey Optimizerを使用すると、ブランドは、カスタマーライフサイクルを通じて、つながりのある、コンテキストに沿ったカスタマージャーニーを提供できます。 チームが大規模なインタラクションをパーソナライズし、顧客の期待とビジネス目標を一致させることができます。このドキュメントでは、Journey Optimizer を効果的に使用する際の重要な役割と責任および使用の開始方法について説明します。

**重要なメモ：** Adobe Journey Optimizer では、特定の責任を持つ個別のロールを定義します。組織の構造に応じて、1 人のユーザーが複数の役割またはすべての役割を実行することができます。

## 役割に基づいたクイックスタートガイド

実装を簡単にするために、Adobe Journey Optimizerでは、専門知識に基づいてタスクを固有の役割に整理しています。 各役割は、シームレスなカスタマーエクスペリエンスを提供するのに必要な基本的なタスクに焦点を当てています。

| 役割 | 主要な責任 | 重要なスキル | 典型的なタスク |
|-------------------|----------------------------------|--------------------------------|-----------------------------------------------|
| **管理者** | 環境のセットアップとアクセス管理 | システム設定、ユーザー管理、セキュリティ | サンドボックスの設定、権限の管理、チャネル設定のセットアップ |
| **データエンジニア** | データ基盤とアーキテクチャ | データモデリング、XDM スキーマ、データ品質 | スキーマとデータセットの作成、データ取り込みの設定、データのライフサイクルの管理 |
| **開発者** | 技術的な実装と統合 | モバイル/Web SDK、API、イベント駆動型アーキテクチャ | SDK の統合、イベントの実装、カスタムアクションエンドポイントの作成 |
| **マーケター** | 顧客体験の設計と実行 | ジャーニー設計、コンテンツ作成、データ分析 | ジャーニーを作成し、パーソナライズされたコンテンツを作成し、キャンペーンを最適化します |

各役割は、Adobe Journey Optimizerの実装の特定のフェーズに対応し、構造化された効率的なデプロイメントプロセスを保証します。

## 実装順序と役割の依存関係

Journey Optimizer の実装が成功すると、通常、このシーケンスに従います。このシーケンスは、役割間の依存関係を反映したものです。

1. **管理者**：環境を設定します\
   管理者は、サンドボックスの設定、アクセス制御の設定、チャネル設定の準備を行うことで、基盤を確立します。 これは、他のチームが機能できるようにするために、最初に行う必要があります。
   * 開発、ステージングおよび実稼動サンドボックスの設定
   * 役割、権限、オブジェクトレベルのアクセス制御（OLAC）の設定
   * チャネル設定（メール、SMS、プッシュ、アプリ内、web、コンテンツカード）を設定
   * サブドメインのデリゲートと IP プールの設定
   * 抑制リストと同意ポリシーの設定

2. **データエンジニア**：データ基盤を作成します。\
   データエンジニアは、パーソナライゼーションを強化するデータインフラストラクチャを構築し、顧客データがシステムに出入りする方法を定義します。
   * 顧客 ID 用の ID 名前空間の作成
   * XDM スキーマ（プロファイル、エクスペリエンスイベント、リレーショナル）の設計
   * データセットを設定して、リアルタイム顧客プロファイルに対して有効にします
   * データ取り込みの設定（バッチおよびストリーミング）
   * 複雑な計算のための計算属性の作成
   * ジャーニーのイベントとデータソースの設定

3. **開発者**：技術統合を実装します\
   開発者は、SDK の統合、イベントの送信、API エンドポイントの構築を行うことで、アプリケーションをJourney Optimizerに結び付けます。 これらの実装により、ジャーニーはトリガーと実行が可能になります。
   * Mobile SDK（iOS/Android）とプッシュ通知設定の統合
   * Web エクスペリエンス用の Web SDKの実装
   * アプリケーションからトリガージャーニーへのイベントの送信
   * 外部システム統合用のカスタムアクションエンドポイントの作成
   * Adobe Experience Platform Assuranceを使用した実装のテスト

4. **マーケター**：顧客体験をデザインし実行する\
   マーケターは、あらゆる基本作業を活用して、すべてのチャネルで、ジャーニーを構築し、コンテンツを作成し、顧客体験を最適化します。
   * セグメント化、CSV アップロードまたはオーディエンスコンポジションを使用したオーディエンスの作成
   * AI アシスタントとテンプレートでパーソナライズされたコンテンツをデザイン
   * イベントとオーディエンスのトリガーを使用したマルチチャネルジャーニーの作成
   * ローンチ前に承認ワークフローでテスト
   * パフォーマンスの監視と、レポートインサイトに基づく最適化

**メモ：**&#x200B;このシーケンスは一般的なものですが、一部のアクティビティが並行して実行されることがあります。例えば、開発者がアプリ統合を行っているときに、データエンジニアがスキーマを設定できます。

## はじめに（役割別）

各役割は、焦点に合わせて調整された特定のタスクから開始されます。これらの最初の手順を完了することで、オンボーディングがスムーズになり、全体的な実装プロセスとの整合性が確保されます。

### マーケター向け {#for-marketers}

すべてのチャネルをまたいで、パーソナライズされた顧客体験を作成することに焦点を当てます。

**使用する主な機能：**

* 複数の方法（セグメント定義、CSV アップロード、オーディエンス構成）でオーディエンスを作成し、セグメントを作成します
* テキストおよび画像生成用の AI アシスタントを使用したコンテンツのデザイン
* ドラッグ&amp;ドロップデザイナーを使用したマルチチャネルカスタマージャーニーの作成
* 送信時間の最適化と競合管理を活用して、エンゲージメントを最大化
* 公開前にコンテンツをテストし、承認ワークフローを使用する
* 統合レポートダッシュボードによるパフォーマンスの監視

**開始：** 事前定義済みのテンプレートを使用して、シンプルなウェルカムジャーニーまたは放棄された買い物かごの回復キャンペーンを作成します。

[マーケター→ーザー向けの基本を学ぶ](path/marketer.md)

### データエンジニア向け {#for-data-engineers}

パーソナライズされたエクスペリエンスを強化するデータ基盤を確立します。

**主な責務：**

* ID 名前空間の作成と ID 解決の設定
* プロファイルおよびイベントデータ用の XDM スキーマの設計（標準およびリレーショナル）
* データセットを設定して、リアルタイム顧客プロファイルに対して有効にします
* バッチおよびストリーミングデータ取り込み用のソースコネクタの設定
* セグメント化を簡素化する計算済み属性の作成
* ジャーニー実行用のイベントとデータソースの設定
* データ品質、ガバナンス、ライフサイクルの管理

**開始：** ID 名前空間を設定し、必須のフィールドグループを持つ最初のプロファイルスキーマを作成します。

[データエンジニア向けの基本を学ぶ→](path/data-engineer.md)

### 管理者向け {#for-administrators}

組織のJourney Optimizer環境を設定および管理します。

**主な責務：**

* 開発、テスト、実稼動用のサンドボックスを作成および管理します
* 標準またはカスタムの役割を使用して役割と権限を設定します
* オブジェクトレベルのアクセス制御（OLAC）を適用してリソースを保護
* メール、SMS、プッシュ、アプリ内、web およびコンテンツカードのチャネル設定の指定
* サブドメインのデリゲートと E メール配信品質のための IP プールの作成
* 抑制リストと許可リストの管理
* 同意ポリシーとデータガバナンスの設定（ヘルスケア/プライバシーシールドを使用）

**開始：** サンドボックスを設定し、基本的な役割と権限を設定してから、チームと協力してチャネル設定を行います。

[管理者向けの基本を学ぶ→](path/administrator.md)

### 開発者向け {#for-developers}

Journey Optimizerをアプリケーションに接続する技術統合を実装する。

**主な責務：**

* Adobe Experience Platform Mobile SDKの統合（iOS/Android）
* Web エクスペリエンスおよび web プッシュ通知用の web SDKの実装
* プッシュ通知の資格情報と証明書を設定
* アプリケーションからトリガージャーニーへのイベントの送信
* カスタムアクションを使用してJourney Optimizerが呼び出す API エンドポイントのビルド
* Web、モバイルおよびその他のサーフェス用のコードベースのエクスペリエンスを実装する
* Adobe Experience Platform Assuranceを使用した実装のテストとデバッグ
* Journey Optimizer API を使用したプログラムによるアクセス

**開始：** モバイルまたは web SDKを統合してから、最初のイベントを実装してジャーニーをトリガーします。

[ 開発者→ーザー向けの基本を学ぶ ](path/developer.md)

## Cross-Role Collaboration

Journey Optimizerを正常に実装するには、次のようなすべての役割でのコラボレーションが必要です。

* **管理者** サンドボックス、権限、チャネル設定を設定して、他の役割を有効にします
* **データエンジニア** は、開発者とマーケターが構築するデータ基盤を提供します
* **開発者** マーケターがジャーニーの開発に使用する技術的トリガーを実装します
* **マーケター** データ品質、機能リクエスト、ユーザーエクスペリエンスに関するフィードバックをすべてのチームに提供します

**ベストプラクティス：** 部門横断的な定期的なミーティングを開催して、優先度に合わせて調整し、進捗を共有し、チーム間でブロッカーに対処します。
