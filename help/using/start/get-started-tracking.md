# Journey Optimizer のトラッキングの基本を学ぶ {#get-started-tracking}

トラッキングを使用すると、キャンペーンの効果を測定し、カスタマーエクスペリエンスを最適化し、メッセージが目的の受信者に確実に届くようにすることができます。Journey Optimizer には、顧客インタラクション、配信パフォーマンス、システムの健全性をキャプチャする包括的なトラッキング機能が用意されています。これにより、プライバシーを適用し、コンプライアンスを維持しながらデータに基づく意思決定を実行できます。

ほとんどのトラッキングは、メッセージとジャーニーを作成する際に自動的に設定されます。高度なシナリオの場合は、カスタム指標の設定、URL パラメーターの設定、外部分析プラットフォームとの統合を行うことができます。ビルトインレポートを通じてトラッキングデータにアクセスするか、書き出して Customer Journey Analytics でより詳細な分析を行うことができます。

>[!BEGINSHADEBOX]

Journey Optimizer で追跡できる内容：

📧 **メールインタラクション** - 開封数、クリック数、リンクパフォーマンス

🌐 **Web 動作** - ページビュー数、クリック数、エンゲージメントパターン数

🛤️ **ジャーニーパフォーマンス** - カスタム指標、ステップイベント数、コンバージョンパス数

📊 **配信品質の健全性** - バウンス率、スパムの苦情、送信者の評判

⚙️ **システム操作** - アラート数、エラー数、カスタムアクションパフォーマンス

>[!ENDSHADEBOX]

## チャネルをまたいだ顧客インタラクションの追跡 {#tracking-by-channel}

Journey Optimizer には、チャネル固有のトラッキング機能が用意されています。各チャネルのトラッキングの設定と使用方法を次に示します。

+++メールトラッキング

メールトラッキングは、メールメッセージを作成する際に自動的に有効になります。Journey Optimizer は、開封数、クリック数、登録解除数をデフォルトで追跡します。追加の設定は必要ありません。

**トラッキングオプションの設定：**

* **トラッキングを有効／無効にする** - メールをデザインする際に、メッセージレベルでトラッキングを制御します。 開封数、クリック数またはその両方を追跡するように選択できます。[詳細情報](../email/message-tracking.md)

* **URL トラッキングパラメーターを設定** - サーフェスレベルでトラッキングパラメーターを設定して、すべてのメールリンクにキャンペーン識別子（utm_campaign、utm_source など）を自動的に追加します。これにより、デジタルエコシステム全体でアトリビューショントラッキングが可能になります。[詳細情報](../email/url-tracking.md)

* **保存されたフラグメントのリンクを追跡** - トラッキングが有効になっているコンテンツからフラグメントを保存した際、他のジャーニーやキャンペーンで再利用すると、そのフラグメントのリンクは追跡されたままになります。[詳細情報](../content-management/save-fragments.md)

* **ミラーページトラッキングを追加** - ミラーページオプションを有効にすると、閲覧者の自動トラッキングを使用する web バージョンのメールが作成されます。[詳細情報](../email/message-tracking.md#mirror-page)

**パフォーマンスの監視：**&#x200B;開封数、クリック数、リンクレベルパフォーマンスなど、キャンペーンレポートとジャーニーレポートでリアルタイム指標を表示します。[キャンペーンレポート](../reports/campaign-global-report-cja-email.md) | [ジャーニーレポート](../reports/journey-global-report-cja-email.md)

+++

+++Web トラッキング

Web トラッキングでは、web の変更に対するユーザーインタラクションを追跡するために明示的な設定が必要です。

**クリックの追跡の設定：**

Web ページを作成する際に、追跡する特定の要素（ボタン、画像、リンク）を選択できます。これにより、追加のコードを必要とせずに、これらの要素のクリックの追跡が可能になります。[詳細情報](../web/monitor-web-experiences.md)

* **クリック可能な要素を追跡** - web パーソナライゼーションでボタン、画像、リンク、任意のインタラクティブな要素を選択します。
* **自動データ収集** - 設定すると、Journey Optimizer はクリックイベント数を自動的にキャプチャし、プロファイルに関連付けます。
* **リアルタイムで監視** - ユーザーインタラクションを追跡し、パーソナライゼーションの有効性を検証します。

**トラッキングデータの表示：**&#x200B;レポートで表示指標、クリックスルー率、要素レベルのパフォーマンスにアクセスします。[キャンペーンレポート](../reports/campaign-global-report-cja-web.md) | [ジャーニーレポート](../reports/journey-global-report-cja-web.md)

+++

+++プッシュ通知トラッキング

プッシュトラッキングは自動的に有効になり、インプレッション数（配信済み）、クリック数（タップ済み）、開封数（アプリを起動済み）をキャプチャします。トラッキング値を最大化するには、プッシュコンテンツでクリック可能な要素を設定します。

**追跡する要素の設定：**

* **本文クリックの動作** - ユーザーが通知をタップした際の動作（アプリを開く、ディープリンクに移動する、web URL を開く）を設定します。各アクションは自動的に追跡されます。[詳細情報](../push/design-push.md#on-click-behavior)

* **アクションボタンを追加** - 各ボタンのアクション（アプリを開く、ディープリンク、web URL）に個別のトラッキングを使用して、最大 3 つのボタン（Android）または複数のボタン（iOS）を含めます。[詳細情報](../push/design-push.md#add-buttons-push)

* **トラッキングを有効にする** - プッシュジャーニーアクティビティまたはキャンペーントラッキング設定でトラッキングが有効になっていることを確認します。[詳細情報](../push/create-push.md#create)

>[!NOTE]
>
>プッシュトラッキングには、Mobile SDK の実装が必要です。 アプリで Adobe Experience Platform Mobile SDK が適切に設定されていることを確認します。[詳細情報](../push/push-configuration.md#integrate-mobile-app)

**エンゲージメントの分析：**&#x200B;クリックスルー率、ボタンパフォーマンスおよび追跡したリンクの詳細をレポートに表示します。[キャンペーンレポート](../reports/campaign-global-report-cja-push.md) | [ジャーニーレポート](../reports/journey-global-report-cja-push.md)

+++

+++アプリ内メッセージトラッキング

アプリ内メッセージは、表示数とユーザーインタラクションを自動的に追跡します。トラッキングの有効性を最大限に高めるために、トリガーとコンテンツを設定します。

**トラッキングの設定：**

* **表示ルールを定義** - トリガー（アプリの起動、画面の読み込み）、頻度ルール、オーディエンス条件を使用して、アプリ内メッセージが表示されるタイミングと場所を設定します。適切な設定により、トリガーされたメッセージと表示されたメッセージの両方の正確なトラッキングを確保できます。

* **追跡する要素を追加** - メッセージコンテンツにボタン、リンク、インタラクティブな要素を含めます。各インタラクションは、詳細なラベルで自動的に追跡されます。

* **表示タイミングを最適化** -トリガーされたメッセージが実際にユーザーに表示される可能性を最大限に高めるために、曜日と時間帯のルールを設定します。

[アプリ内メッセージの設定方法の詳細情報](../in-app/create-in-app.md)

**追跡する内容：** Journey Optimizer は、表示数、ボタンのクリック数、終了数、トリガーされた指標と表示された指標、リンクパフォーマンスを自動的にキャプチャします。[キャンペーンレポート](../reports/campaign-global-report-cja-inapp.md) | [ジャーニーレポート](../reports/journey-global-report-cja-inapp.md)

+++

+++SMS と MMS トラッキング

SMS トラッキングには、最小限の設定が必要です。Journey Optimizer は、メッセージに含まれるリンクを自動的に短縮して追跡します。

**仕組み：**

* **自動リンクトラッキング** - URL ヘルパー機能を使用して、SMS コンテンツに任意の URL を追加します。Journey Optimizer は、追加の設定なしで、リンクを自動的に短縮し、クリック数を追跡します。URL 短縮を使用するには、最初に SMS サブドメインを設定する必要があります。[詳細情報](../sms/sms-subdomains.md)

* **インバウンドメッセージトラッキング** - 受信者からの返信は自動的にキャプチャされるので、双方向の対話と応答パターンを監視できます。[詳細情報](../sms/sms-opt-out.md#sms-native-keywords)

**指標の表示：**&#x200B;レポートでリンククリックデータ、インバウンドメッセージの量、メッセージタイプのパフォーマンスにアクセスします。[キャンペーンレポート](../reports/campaign-global-report-cja-sms.md) | [ジャーニーレポート](../reports/journey-global-report-cja-sms.md)

+++

+++コードベースのエクスペリエンストラッキング

コードベースのエクスペリエンスでは、トラッキングデータを Adobe Experience Platform に送信するための実装設定が必要です。

**前提条件：**

トラッキングが機能する前に、インタラクションイベント（表示数、クリック数）を Adobe Experience Platform に送信するように実装を設定する必要があります。これには以下が必要です。

* Adobe Experience Platform 用に設定されたデータストリームを指定する。[詳細情報](https://experienceleague.adobe.com/docs/experience-platform/edge/datastreams/overview.html?lang=ja)
* Web SDK または Mobile SDK を使用してコードにイベント収集を実装する。
* コンテンツが表示またはクリックされた際に、表示イベント数とインタラクションイベント数を送信する。

[実装の前提条件の詳細情報](../code-based/code-based-prerequisites.md#reporting-prerequisites)

**追跡する内容：**&#x200B;実装すると、任意のデジタルタッチポイント（web サイト、モバイルアプリ、IoT デバイスなど）での表示数、クリック数、クリックスルー率、要素レベルパフォーマンスを追跡します。[キャンペーンレポート](../reports/campaign-global-report-cja-code.md) | [ジャーニーレポート](../reports/journey-global-report-cja-code.md)

+++

+++コンテンツカードトラッキング

コンテンツカードは、ユーザーインタラクションを自動的に追跡します。トラッキング動作を制御するコンテンツと表示ルールを設定します。

**実装方法：**

* **追跡するコンテンツをデザイン** - コンテンツカードにボタンとリンクを追加します。各インタラクティブ要素は、ラベルと URL を使用して自動的に追跡されます。

* **永続性を設定** - コンテンツカードはアプリセッション全体で保持されるので、長期的なエンゲージメントパターンを追跡できます。カードを追跡可能な期間を制御する有効期限ルールを設定します。

* **表示ルールを設定** - カードが表示されるタイミングと場所を定義して、表示数とインタラクションの正確なトラッキングを確保します。

[コンテンツカードの設定方法の詳細情報](../content-card/create-content-card.md)

**エンゲージメントの監視：**&#x200B;複数のセッションをまたいで、表示数、クリック数、クリックスルー率、エンゲージメントパターンを追跡します。[キャンペーンレポート](../reports/campaign-global-report-cja-content.md) | [ジャーニーレポート](../reports/journey-global-report-cja-content.md)

+++

+++ランディングページトラッキング

ランディングページには、追加の設定を必要としないビルトイントラッキングが用意されています。Journey Optimizer は、訪問回数、コンバージョン数、バウンス率を自動的にキャプチャします。

**自動的に追跡する内容：**

* **訪問回数** - リーチを測定する合計訪問回数とユニーク訪問回数
* **コンバージョン率** - フォーム送信、購読の確認、その他の定義済みアクション
* **バウンス率** -やり取りなしで離脱する訪問者の割合
* **パフォーマンスのトレンド** - 指標の進化を示す時系列データ

[ランディングページの設定方法の詳細情報](../landing-pages/create-lp.md)

**パフォーマンスの監視：**&#x200B;訪問パターン、コンバージョン率、バウンス率を時間の経過と共に追跡し、ユーザーがフォームをどのようにやり取りしているかを理解して、改善すべき領域を特定します。[キャンペーンレポート](../reports/lp-report-global-cja.md)

+++

## ジャーニーとキャンペーンアクティビティの追跡 {#journey-campaign-tracking}

チャネルレベルのトラッキングを超えて、全体的なパフォーマンスを測定し、マーケティングイニシアチブをまたいで顧客行動を理解するトラッキングを設定します。

* **カスタム成功指標を定義** - 標準のエンゲージメント指標を超えて、ビジネス目標（購入、新規登録、更新など）に合わせた特定の KPI を設定します。[詳細情報](../building-journeys/success-metrics.md)

* **ジャーニーステップイベントを有効にする** - 顧客がジャーニーを進む際に実行するすべてのアクションの詳細なトラッキングをアクティブ化します。これにより、エントリ／終了ポイント、パスの選択、離脱場所を詳細に把握できます。[詳細情報](../reports/journey-step-events-overview.md)

* **スケジュールを設定** - 送信時間の最適化を設定して、様々なタイミング戦略をまたいでパフォーマンスを追跡し、最適な送信ウィンドウを特定します。[詳細情報](../building-journeys/send-time-optimization.md)

* **カスタムアクションの監視を設定** - 外部システムとの統合のトラッキングを設定して、API 呼び出し、応答時間、エラーパターンを監視します。[詳細情報](../action/reporting.md)

* **カスタムレポートを作成してデータをエクスポート** - カスタマイズされたレポートを作成し、トラッキングデータを外部システムにエクスポートして、より詳細な分析を行います。 [詳細情報](../reports/sharing-overview.md)

* **統合パフォーマンスの表示：**&#x200B;キャンペーンとジャーニーの両方の包括的なレポートにアクセスして、メール、プッシュ、SMS、他のチャネルをまたいでパフォーマンスを比較し、最適な結果をもたらす組み合わせを理解します。[キャンペーンレポート](../reports/campaign-global-report-cja.md) | [ジャーニーレポート](../reports/journey-global-report-cja.md)

## 最適化と決定パフォーマンスの追跡 {#optimization-decisioning-tracking}

Journey Optimizer は、最適化実験、ターゲティング戦略、決定パフォーマンスを自動的に追跡します。適切なデータ収集が確実に行われるように設定を指定します。

### 最適化トラッキングの設定 {#optimization-tracking}

* **キャンペーンとジャーニーの最適化**：

   * 実験を作成する際は、追跡する指標（コンバージョン率、クリック数、カスタムイベント数）を定義します。Journey Optimizer は、各処理のパフォーマンスデータを自動的に収集します。[詳細情報](../content-management/optimization-experimentation.md)

   * 様々なオーディエンスセグメントに異なるコンテンツを配信するターゲティングルールを作成します。Journey Optimizer は、各ターゲットグループのエンゲージメント指標を自動的に追跡するので、セグメントをまたいでパフォーマンスを比較できます。[詳細情報](../content-management/optimization-targeting.md)

* **ジャーニーパスの最適化**：ジャーニーに&#x200B;**最適化**&#x200B;アクティビティを追加し、複数のパスを設定します。 Journey Optimizer は、プロファイルが使用するパスを自動的に追跡し、パフォーマンスを測定します。[詳細情報](../building-journeys/optimize.md)

結果を分析するには：実験レポートでコンバージョン率、統計的有意差、処理間の上昇率を表示するか、ターゲットセグメントをまたいでエンゲージメント指標を比較します。[実験キャンペーンレポート](../reports/campaign-global-report-cja-experimentation.md) | [実験ジャーニーレポート](../reports/journey-global-report-cja-experimentation.md) | [ジャーニーターゲティングレポート](../reports/journey-global-report-cja.md#targeting)

### 決定パフォーマンスの追跡 {#decisioning-tracking}

決定を使用してコンテンツをパーソナライズする際、Journey Optimizer は追加の設定を必要とせずに、決定イベント数、インプレッション数、クリック数を自動的に追跡します。

* **自動イベントキャプチャ** - Journey Optimizer は、プロファイルに対して決定項目が選択されるたびに、決定イベント数を自動的にキャプチャします。
* **インプレッショントラッキング** - メールの場合、インプレッション数は自動的に追跡されます。コードベースのエクスペリエンスの場合、コードに提案表示イベント数を実装する必要があります。[詳細情報](../code-based/code-based-implementation-samples.md#client-side-how)
* **クリックの追跡** - 決定項目のクリック数は、メールで自動的に追跡されます。コードベースのエクスペリエンスには、クリックイベント数の実装が必要です。

>[!NOTE]
>
>**コードベースのエクスペリエンス**&#x200B;で決定を追跡するには、実装で Web SDK または Mobile SDK を使用して、提案インタラクションイベント数（表示数とクリック数）が Adobe Experience Platform を送信する必要があります。[詳細情報](../experience-decisioning/data-collection/schema-requirement.md)

パフォーマンスを監視するには：決定 KPI を表示し、決定項目を比較し、選択戦略を分析し、レポートで AI モデルのパフォーマンスを監視します。[詳細情報](../experience-decisioning/cja-reporting.md)

## トラッキングデータの使用の制御 {#data-governance}

データガバナンスポリシーを使用すると、組織全体での追トラッキングデータの使用方法を制御できます。

* **機密性の高いトラッキングデータにラベル付け** - 追跡する行動データ（健全性コンテンツのクリック数、金融商品のインタラクションなど）にガバナンスラベルを適用して、機密データまたは規制対象データとしてマークします。

* **データ使用を制限** - ラベル付けされたトラッキングデータが特定のチャネルで使用されたり、サードパーティのシステムにエクスポートされたり、特定のパーソナライゼーションシナリオで使用されたりするのを防ぐポリシーを作成します。

* **自動適用** - Journey Optimizer は、ジャーニーやキャンペーンを作成する際にガバナンスポリシーを自動的に確認し、追跡したデータが定義済みポリシーに違反して使用されている場合は公開をブロックします。

データガバナンスにより、GDPR や CCPA などの規制へのコンプライアンスが確保されると同時に、承認された境界内で顧客行動を追跡および分析できます。[詳細情報](../action/action-privacy.md)

## 配信品質とシステムの健全性の監視 {#monitoring-capabilities}

エンゲージメントの追跡を超えて、メッセージがインボックスに届き、システムが最適に実行することを確保するための監視を設定します。

配信品質の監視は、主要なインジケーターを追跡することで、メッセージが受信者のインボックスに確実に届くようにし、送信者の健全な評価を維持するのに役立ちます。

* 定期的に&#x200B;**抑制リストを確認**&#x200B;して、アドレスがブロックされている理由を理解し、リストハイジーンを維持します。 [詳細情報](../reports/suppression-list.md)

* **配信エラーを分析**&#x200B;して、エラーを診断し、是正措置を講じます。[詳細情報](../configuration/email-error-types.md)

* DMARC、SPF、DKIM の&#x200B;**ベストプラクティスに従って**、インボックスへの配置を最大化します。[詳細情報](../reports/deliverability.md)

プロアクティブな監視を設定すると、重要なイベントやシステムの問題に関する通知をリアルタイムで受信できるので、カスタマーエクスペリエンスに影響を与える前にすばやく対応できます。

* **アラートを設定** - ジャーニーエラー、カスタムアクションの失敗、重要な問題に関するリアルタイム通知を設定し、問題にすばやく対応します。[詳細情報](../reports/alerts.md)

* **監査ログを有効にする** - 監査ログをアクティブ化して、コンプライアンスとトラブルシューティングのためにリソースに対するすべてのアクションを追跡します。[詳細情報](../privacy/audit-logs.md)

* **統合を監視** - カスタムアクションのパフォーマンスと外部システムの接続を追跡して、統合の問題を早期に特定します。[詳細情報](../action/reporting.md)
