# 施策でウェーブを使用して送信 {#send-using-waves}

アウトバウンドキャンペーンメッセージの配信を複数のバッチ（ウェーブ）に分割し、経時的にスケジュールできます。 ウェーブ送信は、負荷のバランスを取り、下流システム（コールセンターやランディングページなど）への負担を回避し、特に大量の送信において、配信品質と送信者のレピュテーションをサポートするのに役立ちます。

<!--
>[!CAUTION]
>
>Wave sending applies to **outbound** actions only (Email, SMS, Push, Direct mail).
-->

Journey Optimizerでは、ウェーブの数、サイズ（オーディエンスに対するパーセンテージまたは絶対数）、各ウェーブの実行時間を定義できます。

## 制限とガードレール {#limitations-guardrails}

* ウェーブ送信は、**アウトバウンド**&#x200B;のアクション（電子メール、SMS、プッシュ、ダイレクトメール）にのみ適用されます。
* 少なくとも&#x200B;**2個のウェーブ**&#x200B;を定義する必要があり、最大&#x200B;**10個のウェーブ**&#x200B;を追加できます。
* 2つのウェーブの開始の最小間隔は&#x200B;**30分**&#x200B;です。
* ウェーブ開始は、キャンペーン開始の前または過去にすることはできません。

## ウェーブ送信の設定 {#configure-wave-sending}

キャンペーンでウェーブを送信する方法とタイミングを設定するには、次の手順に従います。

1. アウトバウンドアクション（電子メール、SMS、プッシュなど）を含む[&#x200B; アクションキャンペーン &#x200B;](create-campaign.md)を作成または開きます。

1. キャンペーンの「**[!UICONTROL スケジュール]**」タブで、「**[!UICONTROL キャンペーンアクションをウェーブで配信]**」を選択します。

   ![](assets/campaign-wave-option.png){width="100%"}

   >[!NOTE]
   >
   >**[!UICONTROL キャンペーンアクションをウェーブで配信]** オプションは、キャンペーンの&#x200B;**[!UICONTROL アクション]** タブでアウトバウンドアクションが選択された場合にのみ表示されます。 [詳細情報](campaign-action.md)

1. 送信するウェーブの数を設定します（例：4）。

   >[!NOTE]
   >
   >少なくとも2つの波を定義し、最大10の波を追加できます。

1. 波のサイズとタイミングを定義する方法を以下に説明します。

### 等しい波 {#equal-waves}

デフォルトでは、オーディエンスは同じサイズのウェーブに分割されます。 最初の波の時間をスケジュールし、各波の開始の間に一定の間隔（例えば2時間）を設定します。

![](assets/campaign-equal-waves.png){width="80%"}

>[!NOTE]
>
>2つのウェーブの開始の最小間隔は&#x200B;**30分**&#x200B;です。

次に、後続の波を自動的にスケジュールします（例えば、午前9:00、午前11:00、午後1:00、午後3:00、第3）。

### カスタム配布 {#custom-distribution}

「**[!UICONTROL カスタム分布]**」オプションを選択して、各ウェーブのサイズをオーディエンス全体に対する割合で定義します（例：15%、20%、25%、40%）。

![](assets/campaign-wave-percentage.png){width="80%"}

**[!UICONTROL Numbers]**&#x200B;を選択して、各波のサイズをプロファイルの絶対数として定義します（例：10,000; 50,000）。

![](assets/campaign-wave-numbers.png){width="80%"}

>[!NOTE]
>* パーセンテージを使用する場合、すべての波は合計100%である必要があります。 そうでない場合は警告が表示されます。
>* 数値を使用する場合、システムはカバレッジを検証しません。ウェーブサイズが意図したオーディエンスをカバーしていることを確認してください。 [詳細情報](#faq)

### カスタムスケジュール {#custom-schedule}

**[!UICONTROL 各波のスケジュール]**&#x200B;を選択して、各波の特定の開始日時を定義します。 波の間隔を均等にする必要はありません（例えば、午前9:00、午前11:00、午後5:00、午後8:30）。

![](assets/campaign-wave-custom-schedule.png){width="80%"}

>[!NOTE]
>
>2つのウェーブの開始の最小間隔は&#x200B;**30分**&#x200B;です。

## ユースケース {#use-cases}

Wave送信は、送信するメッセージの数とタイミングを制御するのに役立ちます。これにより、配信品質を向上させ、送信者のレピュテーションを保護し、送信を運用能力に合わせることができます。 以下のシナリオでウェーブを使用することを検討してください。

* **コールセンターまたは応答管理：** 1日または1時間あたりのメッセージ数を制限して、ダウンストリーム チーム（カスタマーケアなど）が応答を処理できるようにします。 例えば、コールセンターのキャパシティに合わせて、1日あたり20通のメッセージを送信します。

  ![](assets/campaign-waves-ex-call-center.png){width="75%"}

* **配信数と配信品質が多い：** 1回のショットで非常に大きなキャンペーンを送信することを避けます。 配信を長期的に分散することで、送信者のレピュテーションを維持し、スパムとしてフラグが立てられるリスクを低減します。

  ![](assets/campaign-waves-ex-high-volume.png){width="75%"}

* **ランプアップ：**&#x200B;新しいプラットフォームまたはIPを使用する場合、徐々にボリュームを増やして（たとえば、最初の波で10%、次に15%、20%など）、徐々にレピュテーションを構築します。

  ![](assets/campaign-waves-ex-ramp-up.png){width="75%"}

## よくある質問 {#faq}

+++ 波のサイズの合計がオーディエンスの合計と等しくない場合はどうなりますか？

* 波のサイズの合計がオーディエンスの&#x200B;**を超える**&#x200B;場合（例えば、100,000人のオーディエンスの最初の波で100,000人をスケジュールする）、最初の波は完全なオーディエンスに送信され、残りの波には送信する残りの1人が残りません。実行されません。
* 合計&#x200B;**がオーディエンスより小さい**&#x200B;場合（例えば、100,000人のオーディエンスに対して合計40,000個のプロファイルを含む4つのウェーブを定義する）、これらのウェーブに含まれるプロファイルのみがメッセージを受信します。 他のオーディエンスは通信を受け取らず、後の波で再試行されません。

+++

+++ 個々のウェーブに異なるセグメントや基準を割り当てることはできますか？

波のサイズとタイミングのみを定義できます。 受信者の選択は、キャンペーン全体で同じです。個々のウェーブに異なるセグメントや基準を割り当てることはできません。

+++
