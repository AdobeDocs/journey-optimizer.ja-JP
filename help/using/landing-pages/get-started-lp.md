# ランディングページの基本を学ぶ {#get-started-lp}

ランディングページは、メール、web サイト、広告、またはその他のデジタル化された場所からユーザーがクリックスルーした後に表示されるスタンドアロン web ページです。

[!DNL Journey Optimizer] を使用すると、ランディングページを作成およびデザインして、ユーザーはオンラインフォームにアクセスし、コミュニケーションやニュースレターなどの特定のサービスをオプトインまたはオプトアウトできます。

➡️ [購読の設定とランディングページの作成について詳しくは、このビデオを参照してください。](#video)

## ランディングページを使用するタイミング {#when-to-use}

ランディングページは次の場合に使用します。

* メールまたはキャンペーン内のリンクから、マーケティングコミュニケーションや特定のサービスまたはニュースレターを顧客が **オプトインまたはオプトアウト** できるようにします
* 通信を送信する前に **同意を収集** し、自動メールでアクションを確認します
* **外で外部ページを作成せずに、ユーザーを** 専用の web フォーム [!DNL Journey Optimizer] にリダイレクトする

## 事前準備 {#prerequisites}

ランディングページを作成する前に、次の設定手順を実行します。

1. [**サブドメインの設定**](lp-subdomains.md) - ランディングページのホスト専用のサブドメインを設定します。
1. [**ランディングページプリセットの作成**](lp-presets.md#lp-create-preset) - サブドメインやランディングページに適用するその他の設定をプリセットで定義します。
1. [**購読リストの作成**](subscription-list.md) （購読のユースケースの場合） – 顧客に特定のサービスを購読または購読解除させる場合は必須です。

## 仕組み {#how-it-works}

ランディングページの作成とデプロイは、次の順序に従います。

1. [**ランディングページの作成と設定**](create-lp.md) - プリセットの選択、プライマリページの設定および必要なサブページの追加を行います。
1. [**ページのデザイン**](design-lp.md) - [!DNL Journey Optimizer] のドラッグ&amp;ドロップエディターを使用して、ページコンテンツとフォームを作成します。
1. [**テストと公開**](manage-lp.md) - ページをプレビューし、フォームの動作をテストしてから公開してライブにします。
1. [**メッセージまたはジャーニー内のリンク**](../email/message-tracking.md#insert-links) - ランディングページ URL をメール、キャンペーンまたはジャーニーアクションに追加して、顧客がリーチできるようにします。

## 主な機能 {#capabilities}

* [!DNL Journey Optimizer]のコンテンツデザイン機能を活用すると、**レスポンシブランディングページ**&#x200B;を簡単に作成できます。
* ユーザーが特定のサービスをオプトインまたはオプトアウトできる購読リストを含め **オプトインおよびオプトアウトフロー** を迅速かつシームレスに設定します。 [詳細情報](lp-use-cases.md#subscription-to-a-service)
* オプトインまたはオプトアウト時に&#x200B;**確認メール**&#x200B;を送信します。[詳細情報](lp-use-cases.md#send-confirmation-email)
