# ランディングページの基本を学ぶ {#get-started-lp}

ランディングページは、メール、web サイト、広告、またはその他のデジタル化された場所からユーザーがクリックスルーした後に表示されるスタンドアロン web ページです。

[!DNL Journey Optimizer] を使用すると、ランディングページを作成およびデザインして、ユーザーはオンラインフォームにアクセスし、コミュニケーションのオプトインやオプトアウトを行ったり、ニュースレターなどの特定のサービスを購読したりできます。

➡️ [購読の設定とランディングページの作成について詳しくは、このビデオを参照してください。](#video)

## ランディングページを使用するタイミング {#when-to-use}

ランディングページは次の場合に使用します。

* 顧客がメールまたはキャンペーン内のリンクからマーケティングコミュニケーションを **オプトインまたはオプトアウト** できるようにします
* 顧客が特定のサービスやニュースレターを **購読または購読解除** できるようにします
* 通信を送信する前に **同意を収集** し、自動メールでアクションを確認します
* **外で外部ページを作成せずに、ユーザーを** 専用の web フォーム [!DNL Journey Optimizer] にリダイレクトする

## 事前準備 {#prerequisites}

ランディングページを作成する前に、次の設定手順を実行します。

1. **サブドメインの設定** - ランディングページのホスト専用のサブドメインを設定します。 [&#x200B; ランディングページのサブドメインの設定 &#x200B;](lp-subdomains.md)
1. **ランディングページプリセットの作成** - サブドメインやランディングページに適用するその他の設定をプリセットで定義します。 [&#x200B; プリセットの作成 &#x200B;](lp-presets.md#lp-create-preset)
1. **購読リストの作成** （購読のユースケースの場合） – 顧客に特定のサービスを購読または購読解除させる場合は必須です。 [&#x200B; 購読リストの作成 &#x200B;](subscription-list.md)

## 仕組み {#how-it-works}

ランディングページの作成とデプロイは、次の順序に従います。

1. ランディングページの **作成と設定** - プリセットの選択、プライマリページの設定および必要なサブページの追加を行います。 [ランディングページの作成](create-lp.md)
1. **ページのデザイン** - [!DNL Journey Optimizer] のドラッグ&amp;ドロップエディターを使用して、ページコンテンツとフォームを作成します。 [&#x200B; ランディングページのデザイン &#x200B;](design-lp.md)
1. **テストと公開** - ページをプレビューし、フォームの動作をテストしてから公開してライブにします。 [&#x200B; ランディングページの管理 &#x200B;](manage-lp.md)
1. **メッセージまたはジャーニー内のリンク** - ランディングページ URL をメール、キャンペーンまたはジャーニーアクションに追加して、顧客がリーチできるようにします。

## 主な機能 {#capabilities}

* [!DNL Journey Optimizer]のコンテンツデザイン機能を活用すると、**レスポンシブランディングページ**&#x200B;を簡単に作成できます。
* **オプトインおよびオプトアウトフロー**&#x200B;を迅速かつシームレスに設定します。
* 購読リストを作成して、ユーザーが&#x200B;**サービスを購読**&#x200B;できるようにします。[詳細情報](lp-use-cases.md#subscription-to-a-service)
* 受信者に、お知らせの受信を&#x200B;**登録解除する機能**&#x200B;を提供します。[詳細情報](lp-use-cases.md#opt-out)
* オプトインまたはオプトアウト時に&#x200B;**確認メール**&#x200B;を送信します。[詳細情報](lp-use-cases.md#send-confirmation-email)
