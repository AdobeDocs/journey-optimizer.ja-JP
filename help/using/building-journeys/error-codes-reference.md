# エラーコードのリファレンス {#error-codes}

Adobe Journey Optimizerでは、標準化されたエラーコードを使用して、ジャーニー、キャンペーンおよびメッセージ設定の問題をすばやく特定し、解決します。 これらのエラーコードを理解することで、トラブルシューティング時間を大幅に短縮し、最適なキャンペーンパフォーマンスを維持するのに役立ちます。

## エラーコード構造について {#error-code-structure}

Adobe Journey Optimizer エラーコードは、コンポーネントと問題のタイプを特定するのに役立つ、一貫した命名パターンに従います。

* **サービスプレフィックス**：エラーを生成したAdobe Journey Optimizer サービス（プッシュ/トランスポートサービスの CJMPTS、メッセージランタイムの CJMRT、ジャーニーオーサリングサービスの CJMAS など）を示します
* **エラー番号**：特定のエラー条件の一意の ID
* **HTTP ステータスコード**：標準 HTTP ステータスコード（400、403、422、500 など）

例：`CJMRT-030012-422` は、エラー番号 030012 および HTTP ステータス 422 （処理不能エンティティ）を持つジャーニーランタイムエラー（CJMRT）を示します。

## エラーコードの場所 {#find-error-codes}

エラーコードは、Adobe Journey Optimizer内の複数の場所に表示されます。

* ジャーニー実行レポートとログ
* Campaign アクティブ化画面
* メッセージ検証の警告
* システム通知およびアラート
* API 応答（REST API 使用時）

エラーが発生した場合は、完全なエラーコードと付随するリクエスト ID をメモします。これらは、トラブルシューティングとサポートエスカレーションに不可欠なものだからです。

## サービス別の一般的なエラーコード {#error-codes-by-service}

### CJMPTS：プッシュサービスエラーとトランスポートサービスエラー {#cjmpts-errors}

これらのエラーは、プッシュ通知の配信およびメッセージトランスポートの操作中に発生します。

| エラーコード | 説明 | 根本原因 | 解決策 |
|------------|-------------|-----------|-----------|
| **CJMPTS-1510-500** | プッシュチャネル送信時の内部サーバーエラー | バックエンドのプッシュ/トランスポートの誤動作、プロバイダーまたはインフラストラクチャのエラー | &#x200B;1. チャネルプロビジョニング設定を確認します <br/>2. プッシュ資格情報が有効であることを確認します <br/>3. 操作をやり直してください <br/>4。 永続的な場合は、Adobe サポートに連絡し、リクエスト ID <br/><br/>**関連ドキュメント**:[&#x200B; プッシュ設定 &#x200B;](../push/push-configuration.md) を提供します |
| **CJMPTS-1023-500** | プッシュ送信/処理中に内部サーバーエラーが発生しました（サードパーティゲートウェイ） | 一時的なクラウドの誤動作または不明なサービスエラー | &#x200B;1. プロバイダー/チャネル構成を確認します <br/>2. サードパーティのゲートウェイのステータスを確認します <br/>3. 数分後に再試行 <br/>4. ログで追加のコンテキスト関連ドキュメ <br/><br/>**トを確認してください**:[&#x200B; プッシュ通知 &#x200B;](../push/create-push.md) |
| **CJMPTS-1310-500** | レンダリングサービスの内部エラー（プレビューまたはライブ送信） | ダウンストリームテンプレートレンダラーが失敗しました。通常、JSON/テンプレート構文の問題が原因です | &#x200B;1. テンプレートの構文と構造を検証する <br/>2. すべてのパーソナライゼーション変数が有効であることを確認します。<br/>3. テストペイロードを使用して問題を特定します <br/>4。 必要に応じてテンプレート <br/><br/>**複雑さを軽減** 関連ドキュメント：[&#x200B; メッセージテンプレート &#x200B;](../content-management/content-templates.md)、[Personalization構文 &#x200B;](../personalization/personalization-syntax.md) |

DNS 伝播の時間を許可（最大 72 時間） <br/><br/>**関連ドキュメント**:[&#x200B; チャネルサーフェス &#x200B;](../configuration/channel-surfaces.md)、[&#x200B; サブドメインデリゲーション &#x200B;](../configuration/delegate-subdomain.md) |
| **CJMRT-110100-500** | ペイロードの内部エラー | バックエンドデータ/設定のバグまたはサポートされていない設定 | 1.操作を再試行します <br/>2. 高度な機能を使用する場合の設定の簡素化 <br/>3. リクエスト ID と正確なペイロードを使用して、Adobe サポートにエスカレ <br/> トします。4. リリースノートで既知の問題を確認してください <br/><br/>**関連ドキュメント**:[ジャーニーのトラブルシューティング &#x200B;](troubleshooting.md) |

UI 関連ドキュメント <br/><br/>**:** メールチャネル [、](../email/get-started-email.md) チャネルサーフェス [&#x200B; で検証メッセージを確認 &#x200B;](../configuration/channel-surfaces.md) ます。 |
| **CJMMAS-2073-422** | メッセージプリセット編集内の処理不可能なエンティティ | 検証エラー、サポートされていないフィールド、または構文が正しくありません | 1.示されている構文/フィールドのエラーを修正します。<br/>2. 正常な構成と比較してください <br/>3。 保存する前に、メッセージ UI の検証を使用します <br/>4. ドキュメント <br/><br/>**関連ドキュメント**&#x200B;[&#x200B; メッセージプリセット &#x200B;](../configuration/channel-surfaces.md)、[&#x200B; メール設定 &#x200B;](../email/email-settings.md) のフィールド要件を確認します。 |

## 一般的なトラブルシューティングアプローチ {#troubleshooting-approach}

エラーコードが発生した場合は、次の体系的なアプローチに従います。

1. **エラーの特定**：完全なエラーコード、HTTP ステータス、付随するメッセージまたはリクエスト ID をメモします。

2. **サービスの検索**：サービスプレフィックス（CJMPTS、CJMRT、CJMMAS、CJMCMP）を使用して、影響を受けるコンポーネントを特定します。

3. **ステータスコードを確認します**。
   * **400 （無効なリクエスト）**：入力データと設定を確認します
   * **403 （禁止）**：権限およびアクセス権を確認してください
   * **409 （競合）**：重複または競合するリソースを検索します
   * **422 （処理不能エンティティ）**：スキーマ要件に照らしてデータを検証します
   * **500 （内部サーバーエラー）**：再試行し、サポートにエスカレーションする可能性があります

4. **最近の変更のレビュー**：最近変更された内容（ジャーニーの更新、新しいキャンペーン、設定の変更など）を考慮します。

5. **ドキュメントを参照**：このガイドに記載されているリンクを使用して、影響を受ける機能の詳細なドキュメントにアクセスしてください。

6. **必要に応じて再試行**:500 系のエラーの場合、数分後の単純な再試行で一時的な問題が解決されることがよくあります。

7. **必要に応じてエスカレーションする**：解決手順に従ってもエラーが解決しない場合は、次の情報でAdobe サポートにお問い合わせください。
   * 完全なエラーコード
   * リクエスト ID （使用可能な場合）
   * 再現手順
   * 関連する設定の詳細

## 一般的なエラーを回避するためのベストプラクティス {#best-practices}

### ジャーニーのアクティベーション前 {#journey-best-practices}

* **すべてのリソースの検証**：参照されるすべてのオーディエンス、データソースおよびカスタムアクションがアクティブであることを確認します
* **十分にテストする**：テストモードを使用して、公開前に問題を特定します（[&#x200B; 詳細情報 &#x200B;](testing-the-journey.md)）
* **権限の確認**：すべてのコンポーネントに対して必要なアクセス権があることを確認します
* **依存関係の確認**：リンクされたすべてのメッセージとコンテンツが公開されていることを確認します

### メッセージの作成時 {#message-best-practices}

* **必須フィールドに入力**：必ずすべての必須フィールドに入力してから保存してください
* **オプトアウトリンクを含める**：すべてのメールバリアントに購読解除リンクを追加します（[&#x200B; 詳細情報 &#x200B;](../privacy/opt-out.md)）
* **パーソナライゼーションの検証**：サンプルプロファイルを使用したすべての動的コンテンツをテストします（[&#x200B; 詳細情報 &#x200B;](../personalization/personalization-build-expressions.md)）。
* **テンプレートを管理可能に保つ**：レンダリングの問題を引き起こす可能性のある複雑すぎるテンプレートを避けます

### キャンペーン管理用 {#campaign-best-practices}

* **オーディエンスデータの検証**：ターゲットオーディエンスが正しく設定され、入力されていることを確認します
* **承認ステータスの確認**：アクティブ化を試みる前に承認要件を理解します（[&#x200B; 詳細情報 &#x200B;](../test-approve/gs-approval.md)）
* **設定を監視**：チャネルサーフェスおよびプリセットが有効かどうかを定期的に確認します
* **DNS 変更の計画**: ドメインの更新時に DNS 伝達に十分な時間を許可する

## サポートを受ける {#getting-support}

このガイドを使用して解決できない永続的なエラーが発生した場合：

1. **情報の収集**：エラーコード、リクエスト ID、タイムスタンプ、再現手順を収集します
2. **システムステータスの確認**：既知のサービスの問題については、[Adobe ステータス &#x200B;](https://status.adobe.com/ja){target="_blank"} を参照してください
3. **ドキュメントを検索**：ソリューションの [Adobe Experience League](https://experienceleague.adobe.com/docs/journey-optimizer.html?lang=ja){target="_blank"} を確認する
4. **コミュニティに参加**:[Adobe Journey Optimizer コミュニティに質問を投稿 &#x200B;](https://experienceleaguecommunities.adobe.com/t5/journey-optimizer/ct-p/journey-optimizer){target="_blank"} ます
5. **Adobe サポートへのお問い合わせ**：関連するすべての詳細を含むサポートチケットを送信します
