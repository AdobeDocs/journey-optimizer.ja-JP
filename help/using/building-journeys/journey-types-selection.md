# ジャーニータイプと選択ガイド {#journey-types-selection}

Adobe Journey Optimizer では、それぞれ異なるエントリメカニズムとビジネスシナリオ向けにデザインされた、次の 4 つのジャーニータイプをサポートしています。このガイドは、違いを理解し、ユースケースに適したタイプを選択するのに役立ちます。

## ジャーニータイプの概要 {#journey-types}

>[!BEGINTABS]

>[!TAB 単一ジャーニー]

**用途：**&#x200B;リアルタイムのイベントトリガーエクスペリエンス

**単一ジャーニー**&#x200B;は、特定のアクション（購入、アプリへのログイン、フォームの送信）が発生した際に個別にトリガーされます。プロファイルはリアルタイムで一度に 1 つずつエントリするので、即時の行動駆動型の応答に最適です。

**最適な用途：**&#x200B;購入後の注文確認、登録時のウェルカムメール、閲覧によりトリガーされた買い物かごの放棄、パスワードリセット通知。

➡️ [イベントの詳細情報](../event/about-events.md) | [サブスクライバーへのメッセージのユースケース](message-to-subscribers-uc.md)

>[!TAB オーディエンスを読み取りジャーニー]

**用途：**&#x200B;オーディエンスセグメントに対するスケジュール済みキャンペーン

**オーディエンスを読み取り**&#x200B;ジャーニーは、Adobe Experience Platform オーディエンスから開始し、すべてのプロファイルにメッセージを一括送信します。このジャーニータイプは、スケジュール済みの大規模な通信に最適です。

**最適な用途：**&#x200B;毎月のニュースレター、ターゲットセグメントに対するプロモーションキャンペーン、製品のお知らせ、季節のマーケティングキャンペーン。

➡️ [オーディエンスを読み取りの詳細情報](read-audience.md) | [オーディエンスの基本を学ぶ](../audience/about-audiences.md)

>[!TAB オーディエンスの選定ジャーニー]

**用途：**&#x200B;オーディエンスメンバーシップの変更に対するリアルタイム応答

**オーディエンスの選定ジャーニー**&#x200B;は、プロファイルが特定のオーディエンスを選定した（または退出した）際にトリガーされます。プロファイルは、条件を満たすとリアルタイムで個別にエントリするので、顧客行動が変化した際に即座のエンゲージメントが可能になります。

**最適な用途：** VIP 層のアップグレード通知、顧客が非アクティブになった際の再エンゲージメント、初回購入に対するお礼メッセージ、顧客が移動した際の地理的ターゲティング。

➡️ [オーディエンスの選定の詳細情報](audience-qualification-events.md) | [オーディエンスの作成](../audience/creating-a-segment-definition.md)

>[!TAB ビジネスイベントジャーニー]

**用途：**&#x200B;複数の顧客に影響を与えるビジネス条件

**ビジネスイベントジャーニー**&#x200B;は、複数のプロファイルに同時に影響を与えるビジネスレベルのイベント（在庫更新、天候アラート、価格変更）によりトリガーされます。これらは、個々のアクションではなく、より広範なビジネス条件に対応します。

**最適な用途：**&#x200B;興味のある顧客への在庫不足アラート、Flash セールのお知らせ、天候に基づくプロモーション、価格低下通知、製品の再入荷アラート。

➡️ [ビジネスイベントの詳細情報](../event/about-creating-business.md) | [エントリ管理](entry-management.md)

>[!ENDTABS]

## 決定ガイド：ジャーニータイプの選択 {#decision-guide}

次の決定ツリーに従って、ユースケースに適したジャーニータイプを選択します。

### 手順 1：ジャーニーをトリガーする内容

* **顧客が特定のアクションを実行する**（購入、クリック、ログイン）→ 手順 2 に進みます
* **時間／スケジュール**（特定の時間に送信または繰り返し）→ **オーディエンスを読み取りジャーニーを使用します**
* **顧客ステータスの変更**（セグメントへの参加／離脱）→ 手順 3 に進みます
* **ビジネス条件**（在庫レベル、価格変更、天候）→ **ビジネスイベントジャーニーを使用します**

### 手順 2：個々の顧客行動トリガー

* **即座のリアルタイム応答が必要ですか？**
   * はい → **単一ジャーニーを使用します**
   * いいえ → スケジュール済み実行によるオーディエンスを読み取りジャーニーを考慮します

### 手順 3：顧客ステータスの変更

* **顧客がセグメントにエントリする際や、セグメントを終了する際に応答する必要がありますか？**
   * はい → **オーディエンスの選定ジャーニーを使用します**
   * いいえ、エントリ時のみ → イベントを含む単一ジャーニーまたはオーディエンスフィルターを含むオーディエンスを読み取りを考慮します

### ユースケース別のクイック選択

| 目標 | 推奨ジャーニータイプ | 理由 |
|-----------|------------------------|-----|
| 購入後に注文確認を送信 | 単一 | 個々のアクションに対する即座の応答 |
| サブスクライバーに毎月のニュースレターを送信 | オーディエンスを読み取り | スケジュール済みバッチ通信 |
| VIP ステータスに達した際に顧客に通知 | オーディエンスの選定 | ステータスの変更に対するリアルタイムの応答 |
| 監視項目の低在庫を顧客に通知 | ビジネスイベント | ビジネス条件により複数の顧客に影響を与える |
| アプリの新規ユーザーを歓迎 | 単一 | 新規登録イベントによりトリガー |
| 非アクティブな顧客に再び関与 | オーディエンスの選定 | 非アクティブなセグメントエントリに応答 |
| ターゲットセグメントに対する季節のプロモーション | オーディエンスを読み取り | オーディエンスに対するスケジュール済みキャンペーン |
| Flash セールのお知らせ | ビジネスイベント | ビジネス決定により複数の顧客に影響を与える |

>[!NOTE]
>
>選択するタイプがわからないですか？イベントベースのエクスペリエンス用の&#x200B;**単一ジャーニー**&#x200B;またはスケジュール済みキャンペーン用の&#x200B;**オーディエンスを読み取りジャーニー**&#x200B;から開始してください。これらは、最も一般的なユースケースを対象としています。

## ジャーニータイプの詳細な比較 {#journey-types-comparison}

次の表を使用して、ジャーニータイプをすばやく比較し、ユースケースに適したタイプを選択します。

| 側面 | 単一ジャーニー | オーディエンスを読み取りジャーニー | オーディエンスの選定ジャーニー | ビジネスイベントジャーニー |
|--------|------------------|------------------------|--------------------------------|------------------------|
| **エントリメカニズム** | 個々のイベントのトリガー | スケジュール済みバッチ | リアルタイムオーディエンスメンバーシップの変更 | ビジネスレベルイベント |
| **エントリタイミング** | イベント発生時のリアルタイム | スケジュール済み（1 回限りまたは繰り返し） | 選定発生時のリアルタイム | ビジネスイベント発生時のリアルタイム |
| **プロファイルエントリ** | 一度に 1 つ | 一度にすべて（バッチ） | 一度に 1 つ | 複数のプロファイルを同時 |
| **ソースをトリガー** | 顧客アクション（購入、クリック、ログイン） | 時間ベースのスケジュール | オーディエンスメンバーシップ （エントリ／終了） | ビジネス条件（在庫、天候、価格） |
| **最適な用途：** | トランザクションメッセージ、行動応答 | マーケティングキャンペーン、ニュースレター | ロイヤルティプログラム、ライフサイクルステージ | 在庫アラート、プロモーション、ビジネス条件 |
| **用途** | 必要な個々のアクションに対する即座の応答 | スケジュールに従った大規模オーディエンスセグメントへの到達 | 顧客ステータスの変更に対する応答 | ビジネスイベントにより複数の顧客に影響を与える |
| **例** | 注文確認、パスワードリセット | 毎月のニュースレター、季節のキャンペーン | VIP のアップグレード、非アクティビティアラート | 低在庫アラート、Flash セール、価格低下 |
| **再エントリ** | 設定可能（プロファイルごとに複数のエントリを許可） | 各プロファイルが実行ごとに 1 回エントリする | 選定イベントごとに設定可能 | 複数のプロファイルが同じイベントで影響を受ける場合がある |
| **データ要件** | トリガーデータを含むイベントスキーマ | Adobe Experience Platform オーディエンス | ストリーミングオーディエンスまたはバッチオーディエンス | ビジネスイベントスキーマ |
