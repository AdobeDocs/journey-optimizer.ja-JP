# ジャーニーの種類と選択ガイド {#journey-types-selection}

Adobe Journey Optimizerでは、4 つのジャーニータイプをサポートしており、それぞれが異なるエントリメカニズムとビジネスシナリオに合わせて設計されています。 このガイドは、違いを理解し、ユースケースに適したタイプを選択するのに役立ちます。

## ジャーニータイプの概要 {#journey-types}

>[!BEGINTABS]

>[!TAB  単一ジャーニー ]

**使用するタイミング：** リアルタイムのイベントトリガーエクスペリエンス

**単一ジャーニー** は、特定のアクション（購入、アプリのログイン、フォームの送信）が発生すると、個別にトリガーされます。 プロファイルはリアルタイムで一度に 1 つずつ入力されるので、すぐに行動に基づく応答を行う場合に最適です。

**最適：** 購入後の注文確認、誰かが購読したときにウェルカムメール、閲覧によってトリガーされた買い物かごの放棄、パスワードリセット通知。

➡️ [ イベントについて学ぶ ](../event/about-events.md) | [ 購読者へのメッセージのユースケース ](message-to-subscribers-uc.md)

>[!TAB  オーディエンスジャーニーの読み取り ]

**使用するタイミング：** スケジュール済みキャンペーンをオーディエンスセグメントに対して

**オーディエンスジャーニーを読み取り** Adobe Experience Platform オーディエンスから開始し、すべてのプロファイルに同時にメッセージを一括で送信します。 このジャーニータイプは、スケジュールされた大規模な通信に最適です。

**最適な対象：** 月刊ニュースレター、セグメントをターゲットにしたプロモーションキャンペーン、製品のお知らせ、季節的なマーケティングキャンペーンなど。

➡️[ オーディエンスを読み取りについて学ぶ ](read-audience.md) | [ オーディエンスの概要 ](../audience/about-audiences.md)

>[!TAB  オーディエンスの選定ジャーニー ]

**使用するタイミング：** オーディエンスメンバーシップの変更に対するリアルタイム応答

**オーディエンスの選定ジャーニー** プロファイルが特定のオーディエンスに適合する（または離脱する）ときのトリガー。 プロファイルはリアルタイムで条件を満たすと個別に入力されるため、顧客の行動が変化した場合でも即座にエンゲージメントを確保できます。

**次の用途に最適：** VIP層のアップグレード通知、お客様の非アクティブ化時の再エンゲージメント、初回購入祝いのメッセージ、お客様の移行時の地域ターゲティング。

➡️[ オーディエンスの選定について学ぶ ](audience-qualification-events.md) | [ オーディエンスの作成 ](../audience/creating-a-segment-definition.md)

>[!TAB  ビジネスイベントジャーニー ]

**用途：** 複数のお客様に影響を与えるビジネス状況

**ビジネスイベントジャーニー** は、複数のプロファイルに同時に影響を与えるビジネスレベルのイベント（在庫の更新、天気アラート、価格の変更）によってトリガーされます。 これらは、個々の行動ではなく、より広いビジネス条件に対応します。

**最適：** 関心のあるお客様への低い在庫アラート、フラッシュセールのお知らせ、天気ベースのプロモーション、価格低下通知、製品在庫アラート。

➡️ [ ビジネスイベントについて学ぶ ](../event/about-creating-business.md) | [ 応募管理 ](entry-management.md)

>[!ENDTABS]

## 決定ガイド：ジャーニータイプの選択 {#decision-guide}

このデシジョンツリーに従って、ユースケースに適切なジャーニータイプを選択します。

### 手順 1：ジャーニーをトリガーするもの

* **お客様が特定のアクションを実行する** （購入、クリック、ログイン）→手順 2 に進む
* **時間/スケジュール** （特定の時間に送信または繰り返し送信）→**オーディエンスを読み取りジャーニーを使用**
* **顧客ステータスの変更** （セグメントへの参加/離脱）→手順 3 に進む
* **経営状況** （株価、価格変動、天候）→**ビジネスイベントジャーニーの使用**

### 手順 2：個々の顧客行動トリガー

* **迅速かつリアルタイムの対応が必要ですか？**
   * 可→**単一ジャーニーを使用**
   * スケジュールされた実行→オーディエンスを読み取りジャーニーを考慮しない

### 手順 3：顧客ステータスの変更

* **顧客のセグメントへのエントリやセグメントからの離脱には応答する必要がありますか？**
   * 可→**オーディエンスの選定ジャーニーを使用**
   * いいえ。「イベントの単一ジャーニー→考慮」または「オーディエンスのオーディエンスを読み取り」フィルターに入力した場合のみ有効です。

### ユースケース別のクイック選択

| 目標 | 推奨されるジャーニータイプ | 理由 |
|-----------|------------------------|-----|
| 購入後に注文確認を送信 | 単一 | 個々の行動に対する即座の対応 |
| 購読者への月次ニュースレターの送信 | オーディエンスを読み取り | スケジュールされたバッチ通信 |
| VIP ステータスに達した場合に通知する | オーディエンスの選定 | ステータスの変更に対するリアルタイムの応答 |
| 監視アイテムの低在庫について顧客に警告します | ビジネスイベント | ビジネス状況が複数の顧客に影響を与える |
| アプリの新規ユーザーを歓迎する | 単一 | 登録イベントによってトリガー |
| 非アクティブな顧客の再エンゲージ | オーディエンスの選定 | 非アクティブなセグメントエントリに応答します |
| ターゲットセグメントへの季節的なプロモーション | オーディエンスを読み取り | オーディエンスに対してスケジュールされたキャンペーン |
| Flash セールのお知らせ | ビジネスイベント | ビジネス上の決定が複数の顧客に影響を与える |

>[!NOTE]
>
>どのタイプを選択するか不明な場合は、 イベントベースのエクスペリエンスの場合は **単一ジャーニー** から開始し、スケジュールされたキャンペーンの場合は **オーディエンスを読み取りジャーニー** から開始します。これらは、最も一般的なユースケースをカバーしています。

## ジャーニータイプの詳細な比較 {#journey-types-comparison}

次の表を使用して、ジャーニーのタイプをすばやく比較し、ユースケースに適したタイプを選択します。

| アスペクト | 単一ジャーニー | オーディエンスジャーニーの読み取り | オーディエンスの選定ジャーニー | ビジネスイベントジャーニー |
|--------|------------------|------------------------|--------------------------------|------------------------|
| **立入メカニズム** | 個々のイベントのトリガー | スケジュール済みバッチ | リアルタイムオーディエンスメンバーシップの変更 | ビジネスレベルのイベント |
| **立入タイミング** | イベント発生時のリアルタイム | スケジュール済み（1 回限りまたは定期的なスケジュール） | 選定が発生した際のリアルタイム | ビジネスイベントの発生時にリアルタイムで通知 |
| **プロファイルエントリ** | 一度に 1 つずつ | すべてを一度に（バッチ） | 一度に 1 つずつ | 複数のプロファイルの同時実行 |
| **トリガー ソース** | 顧客アクション（購入、クリック、ログイン） | 時間ベースのスケジュール | オーディエンスメンバーシップ （入口/出口） | 経営状況（在庫、天候、価格） |
| **最適な対象** | トランザクションメッセージ、行動応答 | マーケティングキャンペーン、ニュースレター | ロイヤルティプログラム、ライフサイクルステージ | 在庫アラート、プロモーション、ビジネス状況 |
| **用途** | 必要な個々のアクションに対する迅速な対応 | スケジュールに沿った大規模オーディエンスセグメントへの到達 | 顧客ステータスの変更への対応 | ビジネスイベントが複数の顧客に影響する |
| **例** | 注文確認、パスワードリセット | 月刊ニュースレター、季節ごとのキャンペーン | VIPのアップグレード、非アクティブ状態のアラート | 低在庫警告、フラッシュセール、価格下落 |
| **再エントリ** | 設定可能（プロファイルごとに複数のエントリを許可） | 各プロファイルは実行ごとに 1 回入力されます | 選定イベントごとに設定可能 | 複数のプロファイルが同じイベントの影響を受ける場合があります |
| **データ要件** | イベントデータを含むトリガースキーマ | Adobe Experience Platform オーディエンス | ストリーミングオーディエンスまたはバッチオーディエンス | ビジネスイベントスキーマ |

| オーディエンスの選定アクティビティ | ✅ | ✅ | ✅ | ✅ |
| ジャンプアクティビティ | ✅ | ✅ | ✅ | ✅ |
| **プロファイル管理** |
| プロファイルの再エントリ | ✅ 設定可能 | 実行ごとに 1 回の ❌ | ✅ 設定可能 | イベントごとの ✅ |
| 名前空間設定 | ✅ Required | ✅ オプション | ✅ Required | ✅ Required |
