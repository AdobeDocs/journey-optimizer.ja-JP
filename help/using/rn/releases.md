# Journey Optimizer リリースサイクル {#releases}

[!DNL Adobe Journey Optimizer] は、定期的に更新され、新機能、改善、修正が提供されます。このページでは、リリースサイクルの仕組みと各リリースフェーズの意味について説明します。これにより、組織で機能が使用可能になるタイミングを簡単に理解できます。

## 継続的配信モデル {#continuous-delivery-model}

[!DNL Adobe Journey Optimizer] は、継続的配信モデルに基づいて動作します。このモデルにより、機能のデプロイメントに対するスケーラブルかつ段階的なアプローチが可能になります。このモデルにより、アドビはイノベーションをより迅速に実現し、ロールアウト中の継続的な安定性とパフォーマンスを確保できます。

>[!NOTE]
>
> [!DNL Adobe Journey Optimizer] は継続的デリバリーを使用するので、組織や地域をまたいで新機能が段階的に表示される場合があります。

このモデルの一部として：

* 新機能と改善は、準備が整い次第、実稼動環境にデプロイされます。

* [**リリースノート**](release-notes.md)&#x200B;は、毎月のリリースの間に&#x200B;_最新の更新内容_&#x200B;の節で更新され、新機能や改善がロールアウトされるたびに発表されるので、リアルタイムで情報を取得できます。

## リリースのタイミングと頻度 {#release-timing}

[!DNL Adobe Journey Optimizer] は通常、毎月のリリースサイクルに従っており、デプロイメントは通常、各月の最終週に行われます。毎月のリリースノートと関連ドキュメントは、リリース週の火曜日に公開されます。プレリリースノートは、リリース週の前の金曜日に公開されます。

>[!TIP]
>
> 各四半期末には、四半期スケジュールまたは依存する製品リリースに合わせて、月末の 2 週間前までにリリースを予測してロールアウトできます。

月間リリースでは新機能と修正の主なセットが導入されますが、継続的デリバリーアプローチにより、準備が整った時点でサイクル間で追加の更新をデプロイできます。リリースノートは、_最新の更新内容_&#x200B;の節でそれに応じて更新され、公開日が記載されます。その月にリリースされたすべての変更は、リリース日に毎月のリリースノートに統合されます。


## リリースパス {#release-paths}

Journey Optimizer の機能は、複雑さ、依存関係、範囲に応じて異なるリリースパスに従います。プラットフォームでは、複数の可用性ラベル（Beta、限定提供、一般提供）を使用しますが、すべての機能がそれらすべてを通過するわけではありません。

一般的なリリースパスは次のとおりです。

* **GA に直接移行** - 一部の新機能と改善は、一般提供（GA）に直接移行されます。
* **LA → GA** - 一部の機能は、一般ロールアウト前に、最初に限定されたオーディエンス（限定提供）に提供されます。
* **Beta → LA → GA** - 大規模な機能や実験的な機能は、テストと検証のためにすべてのフェーズを通じて進みます。
* **Beta → GA** - 特定の安定した Beta 機能は、中間の LA フェーズなしで直接 GA に移行できます。

>[!TIP]
>
> Beta の機能や限定提供の機能への早期アクセスに興味がある場合は、アドビ担当者に連絡して参加オプションについて相談してください。


## 可用性ラベル {#availability-labels}

| **ラベル** | **目的** | **対象** | **主なメモ** |
|------------|-------------|------------------|----------------|
| **Beta** | 早期テストとフィードバック収集。 | アドビのベータプログラムに参加している選択した顧客または組織に限定されます。 | - 実稼動環境での使用を目的としたものではありません。<br> - GA 前に機能やデザインが変更される場合があります。<br> - フィードバックは、最終的な実装を絞り込むのに役立ちます。 |
| **限定提供（LA）** | 検証と監視のための制御されたロールアウト。 | 選択した顧客または環境（例：開発サンドボックス）に対してのみ有効になります。 | - 実稼動環境に対応し、完全にサポートされています。<br> - 一般リリース前にパフォーマンスとスケーラビリティを検証するために使用されます。<br> - アクセスにはアドビの承認が必要です。 |
| **一般提供（GA）** | 完全にサポートされた機能の広範なリリース。 | 実施要件を満たすすべての組織に対してデフォルトで有効になっています。 | - 実稼動環境に対応し、完全にサポートされています。<br> - ライセンスまたは使用権限が適用される場合があります。<br> - 複数の地域をまたいで段階的にロールアウトされる場合があります。 |


## ロールアウトと可用性 {#rollout}

GA 発表後でも、組織または地域をまたいで段階的にロールアウトが行われる場合があります。新機能がすぐに環境に表示されない場合、通常はリリース後数日以内に使用可能になります。

この段階的なロールアウトは、アドビがデプロイメントを完了する前に安定性、パフォーマンス、ユーザーエクスペリエンスを監視するのに役立ちます。


## 常に情報を取得 {#staying-informed}

最新の情報を取得するには：

* 新機能や更新された機能について詳しくは、[**最新のリリースノート**](release-notes.md)&#x200B;を確認してください。
* リアルタイムデプロイメントについて詳しくは、毎月のリリース間の&#x200B;**_最新の更新内容_**&#x200B;の節を確認してください。
* 今後の機能のプレビューについて詳しくは、**プレリリースノート**（使用可能な場合）を監視してください。
* Beta または限定提供アクセスまたは使用権限に関する情報について詳しくは、アドビ担当者にお問い合わせください。

Journey Optimizer 製品リリースに関する&#x200B;**メールや製品内アラート**&#x200B;を登録できます。登録するには：

1. **Adobe Experience Cloud 環境設定**&#x200B;に移動します
1. **通知**&#x200B;の下にある **Journey Optimizer** を見つけます。
1. **新しいリリース**&#x200B;のアプリ内通知およびメール通知を有効にします

## よくある質問 {#faq}

Adobe Journey Optimizer のリリースサイクルに関するよくある質問を以下に示します。

さらに詳細が必要ですか？このページの下部にあるフィードバックオプションを使用して、質問を入力するか、[Adobe Journey Optimizer コミュニティ](https://experienceleaguecommunities.adobe.com/t5/adobe-journey-optimizer/ct-p/journey-optimizer?profile.language=ja){target="_blank"}にアクセスしてください。

+++ Adobe Journey Optimizer のリリースはいつスケジュールされていますか？

[!DNL Adobe Journey Optimizer] は通常、毎月の最終週に更新をリリースします。各四半期末には、クロスソリューションまたはプラットフォーム全体の更新に合わせて、リリースが最大 2 週間早められる場合があります。

+++

+++ 新機能が発表されてもすぐに表示されないのはなぜですか？

一部の GA 機能は、プラットフォームの安定性とパフォーマンスを確保するために段階的にロールアウトされます。機能がすぐに表示されない場合は、地域または組織でのデプロイメントが完了すると表示されます。

+++

+++ Beta、限定提供、GA の違いは何ですか？

* **Beta**：早期テストフェーズ、限定アクセス、フィードバックベース。
* **限定提供（LA）**：最終検証のために制御されたロールアウト。
* **一般提供（GA）**：使用権限のあるすべての顧客に対する完全リリース。

+++

+++ すべての機能が Beta および限定提供（LA）を通じて提供されますか？

いいえ。一部の機能は、その性質と準備状況に応じて、GA に直接リリースされるか、LA にのみにリリースされます。リリースパスは、俊敏性、品質、安定性のバランスを取るために各機能に合わせてカスタマイズされます。

+++

+++ Beta または限定提供プログラムに参加するにはどうすればよいですか？

アクセスは、アドビ担当者からの招待またはリクエストによって行われます。参加することで機能デザインの形成に役立ち、最終リリースでユースケースが考慮されます。

+++

+++ リリースノートはどのくらいの頻度で更新されますか？

リリースノートは継続的に更新されます。月間リリース以外にも、新機能や改善が実稼動環境にプッシュされるたびに、_最新の更新内容_&#x200B;の節が更新されます。
